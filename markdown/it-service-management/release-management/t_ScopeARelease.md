# Define scope of a release in Release Management

Add projects, epics, stories, enhancements, and defects to define the scope of a release.

## Before you begin

Role required: release\_v2\_admin

## Procedure

1.  Navigate to **All** &gt; **Release** &gt; **Releases**.

2.  Click the release number for which you want to define the scope.

3.  On the Release form, navigate to one of the following related lists.

    -   Projects
    -   Scrum Epics
    -   Scrum Stories
    -   SAFe Epics
    -   SAFe Features
    -   SAFe Stories
    -   Defects
    -   Enhancements
    -   Incidents
    -   Problems
    -   Changes
    **Note:** To view the respective work items, ensure that the plugins Agile Development 2.0, Project Portfolio Management, Essential SAFe, and Portfolio SAFe are active.

4.  Add a task to the related list.

    1.  Click **Attach task**.

    2.  Find and locate the required task by clicking the search icon \(\[Omitted image "search-icon.png"\] Alt text: Search icon\).

    3.  Add the task as a release task by clicking the task number.

5.  Click **Submit**.
