# Release key date form

Manage your release key dates for a timeline-oriented release.

## Release key date

For more information about adding key dates to a release, see [Add key dates to a timeline-oriented release](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/it-service-management/digital-product-release/dpr-work-release.md).

For more information about viewing and managing key dates in a release, see [Work on a timeline-oriented release for a single product or service](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/it-service-management/digital-product-release/dpr-work-release.md).

Type of key date. Select one of the options:-   Milestone: An event used to mark the progress toward the overall goal. For example, dev complete.
-   Key event: Any release activity that holds strategic significance.
-   Deadline: The latest date by which a task or activity must be completed. For example, code freeze date.
-   Important date: Any date that holds significance in the release timeline.

</td></tr><tr><td>

Due date

</td><td>

Date and time of the release key date. This date must be between the release start date and end date.

**Note:** You can only add a future dateand it must not be a holiday according to the selected holiday schedule in the release.

</td></tr><tr><td>
