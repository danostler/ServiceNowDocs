# Create change request from a problem

After you investigate and perform a root cause analysis of the Problem, you can understand whether the Problem can be resolved or you should find a temporary workaround for the Problem. If the Problem can be resolved, implement the fix, or the resolution for the Problem.

## Before you begin

Role required: problem\_coordinator or admin

## About this task

You can implement a resolution to a problem by applying a standard change procedure and testing to confirm service recovery. For a normal change, you raise a Request For Change \(RFC\). The Change Advisory Board \(CAB\) needs to approve the RFC before a resolution is applied to the Problem.

## Procedure

1.  Navigate to **All** &gt; **Problem** &gt; **Open** and open a problem record.

2.  On the context menu, perform any one of the following activities.

    |Option|Action to be taken|
    |------|------------------|
    |**Normal change**|Click **Create Normal Change**.|
    |**Emergency change**|Click **Create Emergency Change**.|

    A change request is created to implement the solution for the problem.
