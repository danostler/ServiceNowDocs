# Creating an event rule

You can create an event rule to establish criteria and thresholds for performance, enabling timely alerts and proactive system management.

To create an event rule, perform the following steps.

1.  [Select event rule filters](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/it-service-management/digital-end-user-experience-dex/event-rule-select-filters.md).
2.  [Define alert criteria](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/it-service-management/digital-end-user-experience-dex/define-alert-criteria-event.md).
3.  [Define alert rule name and status](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/it-service-management/digital-end-user-experience-dex/define-alert-name-state.md).

-   **[Select event rule filters](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/it-service-management/digital-end-user-experience-dex/event-rule-select-filters.md)**  
Configure filtering criteria for event rules to target specific devices and users. Use filters to control which events trigger alerts based on operating system, location, department, or custom conditions.
-   **[Define alert criteria](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/it-service-management/digital-end-user-experience-dex/define-alert-criteria-event.md)**  
Specify the event alert criteria by choosing the alert severity, defining thresholds, and specifying conditions for when the alert must be triggered.
-   **[Define alert rule name and status](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/it-service-management/digital-end-user-experience-dex/define-alert-name-state.md)**  
Define alert rule attributes, including the name and state, such as active or inactive.
