# Configure Proofpoint DLP integration settings

Modify the  Proofpoint DLP  integration default system properties.

## Before you begin

Role required: sn\_dlir.admin

## Procedure

1.  Navigate to **Proofpoint DLP Integration** &gt; **Settings**.

2.  Modify the following settings as required.

Field

</th><th align="left" id="d50485e74">

Description

**Maximum Timeout for HTTP calls sn\_pp\_dlp.maximum\_http\_time\_out**

</td><td>

Maximum Timeout for HTTP calls \(in seconds\).-   Type: integer
-   Default value: 30


</td></tr><tr><td id="d50485e98">

**Send DLP Incident updates back to Proofpoint sn\_pp\_dlp.incident\_update**

</td><td>

This property enables restricting any synchronization from your ServiceNow instance to Proofpoint DLP incidents. -   Type: Boolean
-   Default value: true


</td></tr><tr><td id="d50485e122">

**Level of reporting the logs generated by the integration sn\_pp\_dlp.logging.verbosity**

</td><td>

The level of reporting logs generated by the integration.

</td></tr></tbody>
</table>3.  Click **Save**.


## Result

Your modified integration settings are applied in the next polling interval as defined in the profile.
