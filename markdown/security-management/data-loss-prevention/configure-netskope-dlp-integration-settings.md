# Configure Netskope DLP integration settings

Modify the  Netskope DLP  integration default system properties.

## Before you begin

Role required: sn\_dlir.admin

## Procedure

1.  Navigate to **Netskope DLP Integration** &gt; **Settings**.

2.  Modify the following settings as required.

    |Field|Description|
    |-----|-----------|
    |Should downloading the violating file of the reported incident be allowed sn\_ns\_dlp.enable\_file\_download.|Option to choose whether you want to enable download of the file. By default, this option is enabled.|
    |Maximum Timeout for HTTP calls, in seconds sn\_ns\_dlp.maximum\_http\_time\_out.|Maximum Timeout for HTTP calls, in seconds. This field is automatically set to 30.|
    |Maximum number of retries to be performed for API failures. sn\_ns\_dlp.maximum\_retry.|Maximum number of retries to be performed for API failures. The default value is set as 3.|
    |Level of reporting the logs generated by the integration sn\_ns\_dlp.logging.verbosity.|Define the level of reporting logs generated by the integration. This field is automatically set to **info**.|

3.  Click **Save**.


## Result

Your modified integration settings are applied in the next polling interval as defined in the profile.
