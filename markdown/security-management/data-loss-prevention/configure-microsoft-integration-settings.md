# Configure Microsoft DLP IR integration settings

Modify the  Microsoft DLP IR  integration default system properties.

## Before you begin

Role required: sn\_dlir.admin

## Procedure

1.  Navigate to **Microsoft DLP Integration** &gt; **Settings**.

2.  Modify the following properties as required.

<table id="table_bs4_qml_nwb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Maximum Timeout for HTTP calls \(sn\_pp\_dlp.maximum\_http\_time\_out\)

</td><td>

Maximum Timeout for HTTP calls \(in seconds\).-   Type: integer
-   Default value: 30


</td></tr><tr><td>

Level of reporting the logs generated by the integration \(sn\_pp\_dlp.logging.verbosity\)

</td><td>

The level of reporting logs generated by the integration, meaning the name of the type of information. You can also update the value to the following options:-   **error**
-   **warn**
-   **info**
-   **debug**
-   **off**
By default, the value is **info**.

</td></tr><tr><td>

Destination of reporting the logs generated by the integration \(sn\_pp\_dlp.logging.destination\)

</td><td>

The destination for the logs that are generated by the integration. You can also update the value to the following options:-   **db**
-   **file**
-   **none**
By default, the value is **db**.

</td></tr></tbody>
</table>3.  Click **Save**.


## Result

Your modified integration settings are applied in the next polling interval as defined in the profile.
