# View Security Incident Response Premium KPIs dashboard

With this dashboard, security managers can track and view the volume, performance, and progress of security incidents from initial analysis/detection to containment, eradication, and recovery.

## Before you begin

In the current release, the following list of reports is not supported in the Security Incident Response Premium KPIs dashboard:

-   Process by state - Workbench is not available
-   Process by Age - Workbench is not available
-   Security incidents open for more than 30 days by assignment group and state - HeatMap visualisation is not available.
-   Security incidents not updated for more than 30 days by assignment group and state - HeatMap visualisation is not available.
-   Security incidents with assignee that is not active - HeatMap visualisation is not available.

Role required: sn\_si.analyst \(Read\), sn\_si.manager or sn\_si.admin \(write\)

## Procedure

1.  Navigate to **All** &gt; **Workspaces** &gt; **Security Incident Response**.

2.  Click **SIR Dashboards** icon, displayed on the left side of the workspace home page.

3.  Select **Security Incident Response Premium KPIs** from the drop down list.

    \[Omitted image "security-incident-mgmt-premuim-kpis-dashboard.png"\] Alt text: Security Incident Response Premium KPIs dashboard

4.  View the Security Incident Response Premium KPIs dashboard for further action.

    For detailed information on Security Incident Response Premium KPIs dashboard, see [Security Incident Management Premium dashboard](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/security-management/security-incident-response/si-mgmt-premium-dashboard.md)
