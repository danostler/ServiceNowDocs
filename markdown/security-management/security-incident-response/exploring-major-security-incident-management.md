# Exploring Major Security Incident Management

Explore Major Security Incident Management to understand how the application works in terms of tracking high priority critical security incidents, which are identified as a threat to the organization.

Use this section to learn the key features, various roles involved in working with major security incident records. This section also helps you understand on how to get started with the application, prerequisites, and plugins required to work with the application.

-   **[Major Security Incident Management](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/security-management/security-incident-response/major-security-incident-management.md)**  
Track and manage various activities that are typically part of resolving a major security incident through Major Security Incident Management. Through an intuitive workspace, incident managers and those working on an incident can propose and promote incidents to major incidents, track major security incident activities, and easily collaborate with colleagues.
-   **[Get started with MSIM](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/security-management/security-incident-response/get-started-with-msim.md)**  
Review the following information before you start working with Major Security Incident Management.
-   **[Checklist for MSIM setup](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/security-management/security-incident-response/msim-checklist.md)**  
Before using the ServiceNow® Major Security Incident Management \(MSIM\) application, download the application from the ServiceNow® Store.
-   **[Major Security Incident Management roles](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/security-management/security-incident-response/msi-roles.md)**  
Assign roles to ensure that users can perform all necessary actions within the MSIM application.
