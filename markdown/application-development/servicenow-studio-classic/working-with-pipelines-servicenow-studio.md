# Pipelines in ServiceNow Studio

Use pipelines in ServiceNow Studio to automate the propagation and installation of applications from one instance to another. Admins manage deployment requests in the App Engine Management Center \(AEMC\).

Pipelines are powered by the ServiceNow CICD spoke, which automates processes such as publishing applications to the Application Repository, installing them on target instances, running ATF tests, and running instance scans.

**Note:** Deploying scoped apps using pipelines in App Engine Management Center \(AEMC\) is a premium feature that might require an additional license. For more information, see [Installing ServiceNow Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/servicenow-studio-classic/installing-servicenow-studio.md).

Admins use pipelines to move app changes through instances. For more information about configuring pipelines, see [Configure Pipelines and Deployments](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/app-engine-management-center/config-p-and-d.md).

With the App Engine Management Center \(AEMC\) installed, admins can manage deployment requests created when a developer selects **Deploy** in ServiceNow Studio to initiate the pipeline deployment process. For more information, see [Managing deployments using Pipelines and Deployments](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/app-engine-management-center/manage-deployments-using-p-and-d-aemc.md).

**Note:** If your app was created in ServiceNow IDE and converted to Fluent, you must switch from ServiceNow Studio back into the ServiceNow IDE for the publishing and deployment process.
