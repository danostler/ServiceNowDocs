# Opening files in your preferred editor

Many types of metadata that you can access in ServiceNow Studio open in builders. Builders are specialized tools that provide an intuitive and efficient interface for editing metadata records primarily associated with tables extending sys\_metadata. You can choose whether to edit files in a builder or in the classic UI16 experience.

File types with an associated builder open in the builder by default. However, you can choose to open your files in either the builder or the UI16 experience with tables and lists.

When you open an app, select the file type you want to open. Use the more options icon \[Omitted image "sn-studio-more-options-icon.png"\] Alt text: next to the individual file you want to open to select **Open with**. Each file that has an associated builder has this option, and the options vary by file type.

In this example, the Performance Management application is open, and the Flow file type is opened. Next to the individual flow file, you can use the more options icon \[Omitted image "sn-studio-more-options-icon.png"\] Alt text:to choose whether you want to open the flow in Workflow Studio or in the UI16 form.

\[Omitted image "sn-studio-open-with.png"\] Alt text: You can open an individual flow using Flow Designer or the UI16 experience.

For more information about the builders associated with certain file types, see [ServiceNow Studio Navigator panel taxonomy](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/servicenow-studio-classic/servicenow-studio-file-navigator-taxonomy.md).
