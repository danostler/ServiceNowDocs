# ServiceNow Studio tutorial part 1: Begin with an MVP

In this section, you will build a minimum-viable-product \(MVP\) using ServiceNow Studio. The MVP will use a custom data model, record producer, and a custom workspace. You will also use Workflow Studio to structure the MVP's workflow.

-   **[Create an app in ServiceNow Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/servicenow-studio-classic/sns-z-tut-create-an-app-in-sns.md)**  
Build an app quickly in ServiceNow Studio, the new, innovative environment for development on the ServiceNow AI Platform.
-   **[Create the data model](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/servicenow-studio-classic/sns-z-tut-create-data-model.md)**  
Define the data for the Marketing Services application in ServiceNow Studio.
-   **[Build the marketing design request form](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/servicenow-studio-classic/sns-z-tut-build-marketing-design-request-form.md)**  
Edit the form view of the Marketing Design Request table.
-   **[Create the Marketing department record producer](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/servicenow-studio-classic/sns-z-tut-create-marketing-dept-record-producer.md)**  
Create a record producer for the Marketing department using ServiceNow Studio.
-   **[Create the fulfiller workspace](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/servicenow-studio-classic/sns-z-tut-create-fulfiller-workspace.md)**  
Workspaces are a great option for departments. Workspaces can be tailored specifically for department needs and can simplify interacting with the platform.
-   **[Set up the Branding department approvers](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/servicenow-studio-classic/sns-z-tut-set-up-branding-dept-approvers.md)**  
Set up the Branding department approvers on the platform.
-   **[Submit a request](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/servicenow-studio-classic/sns-z-tut-submit-request.md)**  
Submit a request in the Marketing Services workspace to test its functionality.
-   **[Add a workflow with Workflow Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/servicenow-studio-classic/sns-z-tut-add-workflow-flow-des.md)**  
Use Workflow Studio to build a workflow to standardize the way that the department processes requests. Configure the workflow to execute whenever a department request is submitted. This workflow will request an approval from a group of users designated as Department Request Approvers.
