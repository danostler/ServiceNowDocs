# Instance-specific deployment user roles

Assign roles that enable non-admin users to install or upgrade all applications in specific instances. You delegate these tasks by manually assigning specific user roles to an instance.

For example, you can assign user roles to Change Management personnel that enable them to perform application installations in non-production \(development or QA\) instances.

**Important:** Think carefully about who and in what instances you assign installation and upgrade user roles. You may only want to assign these roles to non-system administrators in non-production instances \(for example, Developer or QA\), but decline to do so in production instances. Given the potential impact of installing and upgrading applications, you may want to leave the responsibility for installations and upgrades in production instances to a system administrator.

To learn more about managing per-user subscriptions, see  and contact your account representative.
