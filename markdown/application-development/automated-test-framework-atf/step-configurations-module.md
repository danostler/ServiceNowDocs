# Step configurations

Step configuration records define how each type of step behaves.

-   **[Automated Test Framework Step Config record](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/automated-test-framework-atf/atf-step-config-record.md)**  
The step config record controls how a test step of this type behaves.
-   **[Examples of step config field values](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/automated-test-framework-atf/atf-step-config-xmpls.md)**  
Examples of where the system displays values assigned to some of the step config fields.
-   **[Step execution scripts](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/automated-test-framework-atf/atf-config-script.md)**  
In a step configuration record, the step execution script field determines what a step with this configuration does when it runs.
-   **[Step description generation script](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/automated-test-framework-atf/atf-config-desc-script.md)**  
In a step configuration record, the step description generation script field determines the step description that the system generates when a step of this type is added to a test.
-   **[Add output variables to scripted steps](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/automated-test-framework-atf/scripting_atf.md)**  
Execute the following steps to add additional outputs in Run Server Side Script and Custom Scripted StepConfig test steps.
