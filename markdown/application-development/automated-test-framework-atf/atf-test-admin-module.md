# Administration

The Administration module contains forms for configuring and managing the automated test framework.

-   **[Properties](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/automated-test-framework-atf/atf-admin-properties.md)**  
On the Properties form, you can set parameters that control how the system executes automated tests and test suites.
-   **[Step configurations](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/automated-test-framework-atf/step-configurations-module.md)**  
Step configuration records define how each type of step behaves.
-   **[Test templates](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/automated-test-framework-atf/test-templates-module.md)**  
The Test Templates module opens a list of available templates. From this module, you create, view, and edit test templates.
-   **[Step configuration categories](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/automated-test-framework-atf/step-configuration-categories-module.md)**  
The Step Configuration Categories module opens a list of records specifying the step categories on the Add Step dialog. From this module, you can add, delete, and edit these categories.
-   **[Table cleanup](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/automated-test-framework-atf/table-cleanup.md)**  
The Table Cleanup module opens a list of records specifying the retention policies for test result and test suite result tables and the records within them. From this module, you can view and modify these policies.
-   **[Step environments](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/automated-test-framework-atf/step-environments-module.md)**  
A test step environment specifies where the step executes \(for example, server versus browser\). In this release, custom step configs can use only the Server-Independent environment.
