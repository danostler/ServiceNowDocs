# Workflow for creating automations in Creator Studio

Provide your organization's fulfillers with guidance when addressing records generated from the app by adding automated playbooks to it.

## Before you begin

**Note:** Because playbooks are based on forms, you must publish a form \(mark it as ready\) before you use it to create a playbook. For more information, see [Publish a form for your app in Creator Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/creator-studio/creator-studio-publish-form.md).

To work with automations, you must be given permission to work on the app.

## Procedure

1.  Create an app.

    If you have an existing playbook that's close to what you want to create, you can duplicate and edit it, which may be faster than creating an entirely new playbook. For more information, see [Copy an automated playbook in Creator Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/creator-studio/creator-studio-duplicate-automation.md).

2.  Add an automated playbook to the app.

    For more information, see [Add an automated playbook to an app in Creator Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/creator-studio/creator-studio-add-automation.md).

3.  Add activities to the playbook.

    For more information, see [Add activities to an app's playbook in Creator Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/creator-studio/creator-studio-add-activities-automation.md).

4.  Create different branches of the playbook by adding decisions to it.

    For more information, see [Add a decision to an app's playbook in Creator Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/creator-studio/creator-studio-add-decision-automation.md).

5.  Create any parallel processes to run at the same time in the playbook.

    For more information, see [Add a parallel process to an app's playbook in Creator Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/creator-studio/creator-studio-decision-parallel-process-add.md).

6.  Make any additional changes to the playbook's trigger.

    For more information, see [Edit the trigger for a playbook in Creator Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/creator-studio/creator-studio-add-trigger-automation.md).

7.  Make any additional changes to the playbook settings.

    For more information, see [Edit a playbook in Creator Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/creator-studio/creator-studio-edit-automation.md).

8.  Activate the playbook when it's ready.

    For more information, see [Activate a playbook in Creator Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/creator-studio/creator-studio-activate-automation.md).
