# Preview how an app's records appear

You can preview and interact with the submitted records generated by your app directly in Creator Studio, which mimics what the fullfiller sees in the workspace category for the app.

## Before you begin

You can customize how the record will appear before testing it in a preview. For more information, see [Change the layout of an app's record in Creator Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/creator-studio/creator-studio-work-with-record-details.md).

You can create a submitted record to preview by testing the form with the **Try it** button. Testing the form records your answers, runs any associated playbooks, and generates a new record with the results of the automation. For more information, see [Test one of your forms for an app](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/creator-studio/creator-studio-try-it.md).

To preview how an app's submitted records will appear in the workspace configuration, you must be given permission to work on the app.

## Procedure

1.  Go to **All** &gt; **App Engine** &gt; **Creator Studio** to see all the apps on the Creator Studio home page.

2.  Open the app that contains the workspace configuration that you want to edit.

3.  Select the **List configurations** section in the application header.

4.  Select a filtered list that contains a record, such as the Open records list.

5.  Preview a record by selecting it in the canvas.

    \[Omitted image "crs-workspace-open-list.png"\] Alt text: Select the record number in the list to interact with it in Creator Studio

    The record opens in Creator Studio.

6.  Play with the record to see how its parts will appear.

    For example, you can select the record's **Approvers** tab to view how its approvers list will appear.

    If you're previewing a record that was generated using the **Try it** button, the form's questions \(which are catalog variables\) appear on the **Details** tab. Results of any playbooks appear on the **Automations** tab.

    \[Omitted image "crs-wksp-preview-record-nav.png"\] Alt text: Request App Workspace record preview within Creator Studio


## Result

If you don't like what you see, you can tweak the record's appearance. Find out how in [Change the layout of an app's record in Creator Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-development/creator-studio/creator-studio-work-with-record-details.md).
