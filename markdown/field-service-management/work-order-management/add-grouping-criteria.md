# Add grouping criteria

Define criteria to group similar work orders for a work plan.

## Before you begin

Role required: sn\_fsm\_planned\_wm.planned\_work\_admin

Ensure that you have selected **Enable grouping** for a work plan.

## About this task

Work order grouping consolidates multiple individual work orders into fewer, grouped work orders based on configured criteria such as location, floor, or equipment type. Instead of generating one work order per asset, one work order is created for a group of assets. Every work order will have corresponding tasks and the list of related tasks of that work order.

For example, consider a plan for maintenance of 100 HVAC systems in a large office building with 10 floors. If grouping is not enabled, one work order is generated for each asset. This generates 100 work orders for the maintenance plan.

When grouping is enabled, one work order is generated for all the assets that meet the specified condition. For example, you can specify the grouping condition as **Floor** to group all the HVAC units by the floor number. One work order is generated for all the HVAC units of a floor. So, 10 work orders are generated by grouping the assets.

## Procedure

1.  Navigate to **All** &gt; **Planned Work Management** &gt; **Work Plans**.

2.  Open the work plan that you want to add the grouping criteria.

3.  In the **Work Plan Group Criteria** related list, select New.

4.  On the form, fill the fields.

    |Field|Description|
    |-----|-----------|
    |Table|Specifies the name of the table for which the associated work plan is created. The value is auto-populated with the table name specified in the conditions of the work plan.|
    |Plan|Specifies the name of the plan for which the grouping condition applies. The value is auto-populated with the name of the work plan.|
    |Group by|Select the field by which you want to group the work orders.|
