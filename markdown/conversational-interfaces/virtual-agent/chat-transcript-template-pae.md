# Conversation transcript template

The conversation transcript .txt file contains information used to troubleshoot individual conversations.

Data element

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Conversation ID \(sys\_id\)

</td><td>

Unique identifier of the conversation, autogenerated by the Virtual Agent.

</td></tr><tr><td>

Conversation Start Time

</td><td>

Start time of the conversation.

</td></tr><tr><td>

Channel

</td><td>

Channel from which the conversation was initiated. For example, the Slack application, Microsoft Teams, and so on.

</td></tr><tr><td>

User Name

</td><td>

Name of the user who initiated the conversation.

</td></tr><tr><td>

User Status

</td><td>

Status of the user in the ServiceNow AI Platform® instance. For example, Active or Inactive.

</td></tr><tr><td>

Bot Response

</td><td>

Control that displays Virtual Agent responses in a conversation.

</td></tr><tr><td>

User Input

</td><td>

Control that prompts and captures information from the user in a conversation.

</td></tr><tr><td>

Message Date and Time

</td><td>

Timestamp of individual messages.

</td></tr><tr><td>

FDIH logs

</td><td>

Action or Subflow-   Id: sys\_id of the action or subflow.
-   Inputs: Input passed to the action or subflow.
-   Outputs: Output received from the action or subflow.

</td></tr><tr><td>

Topic Block

</td><td>

-   Inputs: Input passed to the topic block.
-   Outputs: Output received from the topic block.
