# Failure mode form

The following table describes the field values for the Failure mode form.

<table id="table_cr2_nqb_1gc"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Name

</td><td>

Name for the failure mode that you’re creating.

</td></tr><tr><td>

Description

</td><td>

Description for the failure mode that you’re creating.

</td></tr><tr><td>

Effect

</td><td>

Consequence of the failure on the system, process, or product.

</td></tr><tr><td>

Functional location

</td><td>

Work area where the failure has occurred.

</td></tr><tr><td>

Parent

</td><td>

Parent failure mode that the failure has been inherited from.

</td></tr><tr><td>

Type

</td><td>

Type of failure. Options are:-   Failure
-   Failure mode

</td></tr><tr><td>

Severity

</td><td>

A numerical rating \(1-10\) indicating how serious the effect of the failure is. Higher values mean more severe consequences.

</td></tr><tr><td>

Occurrence

</td><td>

A numerical rating \(1–10\) estimating how frequently the failure is likely to occur.

</td></tr><tr><td>

Detection

</td><td>

A numerical rating \(1–10\) indicating how likely it is that the failure is detected before it causes harm. Lower values mean better detectability.

</td></tr><tr><td>

Code

</td><td>

A unique identifier or reference number for the failure mode.

</td></tr><tr><td>

Related state

</td><td>

The operational condition of the system when the failure occurs. Options are:-   Running
-   Stopping
-   Stopped
-   Starting

</td></tr><tr><td>

Planning status

</td><td>

Status of mitigation or maintenance planning. Options are:-   Planned
-   Unplanned
-   Excluded

</td></tr></tbody>
