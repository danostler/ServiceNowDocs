# Managing Entity View Action Mapper

Optimize data presentation and interaction by defining data sources, creating Entity View Action Mapper \(EVAM\) configuration bundles, and setting up EVAM view templates to structure and display entity data effectively.

## Overview of managing EVAM

-   [Define a data source](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/entity-view-action-mapper-evam/define-evam-datasource.md)

    Specify a data source.

-   [Define an EVAM configuration bundle](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/entity-view-action-mapper-evam/define-view-configuration-bundle.md)

    Create a view configuration to combine conditions, database fields, and declarative actions with an associated view template.

-   [Define an EVAM view template](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/entity-view-action-mapper-evam/define-evam-template.md)

    Configure view templates to customize data display for users, mapping fields from the view configuration to the component.
