# User roles installed with Live Feed

The Live Feed plugin includes the following user roles.

To learn more about managing subscriptions, see [Managing per-user subscriptions in Subscription Management](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-administration/managing-user-subscriptions-v2.md) and contact your account representative.

**Parent Topic:**[Components installed with Live Feed](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/live-feed/r_InstalledWithLiveFeed.md)

## live\_feed\_admin

Can manage Live Feed functions. Users with the live\_feed\_admin role can set up table notifications for non-task tables, limit Live Feed access by role, and manage Live Feed message content.

### Contains Roles

List of roles contained within the role.

None.

### Groups

List of groups this role is assigned to by default.

None.

### Special considerations

**Note:** Avoid granting an admin role when more specialized roles are available.

## chat\_admin

Can manage chat functions \(if the Chat plugin is activated\). Users with the chat\_admin role can change chat room message read access and define chat room access rights.

### Contains Roles

List of roles contained within the role.

None.

### Groups

List of groups this role is assigned to by default.

None.

### Special considerations

**Note:** Avoid granting an admin role when more specialized roles are available.
