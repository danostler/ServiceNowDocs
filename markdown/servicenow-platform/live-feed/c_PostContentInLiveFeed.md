# Post content in Live Feed

In Live Feed, you can post new messages and replies to existing messages for all users in the feed. You can also send a reply message to a team or record.

**Important:**

Core UI is required to use Live Feed. Live Feed is not supported in Next Experience. If you turn on Next Experience, Live Feed is automatically disabled and will no longer work. See Next Experience UI for more information.

While Live Feed has been discontinued, we are always exploring new ways to enhance our services. Stay tuned for future updates and improvements.

## Attachments in feeds

In addition to entering text, you can attach files, such as images, documents, and spreadsheet files in any ServiceNow supported file type to Live Feed messages.

Administrators can manage attachment file settings, such as supported file types and maximum attachment file size, by editing system properties.

You can also include links to user-defined URLs and knowledge articles contained in your instance's knowledge base.

**Note:** The functionality described here applies to HTML5-compliant browsers, such as Chrome, Firefox, Safari, and Internet Explorer 10 and above. Significant differences while using different browsers are highlighted.

-   **[Post a Live Feed message](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/live-feed/t_PostLiveFeedMessages.md)**  
You can post a message to open a new feed.
-   **[Reply to a Live Feed message](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/live-feed/t_ReplyToAMessage.md)**  
To participate in an existing thread of a feed, you can compose and submit a reply message. By default, the message is visible to all members of the feed.
-   **[Attach a file to a post or reply](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/live-feed/t_AttachAFileToAPostOrReply.md)**  
You can attach files to a post by dragging a file into a post or pasting an image from the clipboard.
-   **[Copy an image from the clipboard](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/live-feed/t_CopyAnImageFromTheClipboard.md)**  
In addition to adding files to a message, you can also copy images to the clipboard and paste them into a message or reply.
-   **[Add a link to a post or reply](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/live-feed/t_AddALinkToAPostOrReply.md)**  
Add or modify a link to a Live Feed post.
-   **[Add a knowledge article link to a post or reply](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/live-feed/t_AddKnowledgeArtLinkPostRep.md)**  
Use the ID of a knowledge article to post it as a reply to a Live Feed conversation.
-   **[Add a poll to a message](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/live-feed/t_AddAPollToAMessage.md)**  
When you are posting a message, you have the option of creating a poll in order to record your viewers' opinions. As users take the poll, their feedback statistics are displayed within the feed.
-   **[Add mentions to a message](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/live-feed/add-mentions-to-message.md)**  
An @mention is any posted update that contains @username anywhere in the body of the message.
-   **[Like a message](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/live-feed/t_LikeAMessage.md)**  
Like a message to provide feedback to the author.
-   **[Delete a message](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/servicenow-platform/live-feed/t_DeleteAMessage.md)**  
Users can delete any of their own posts or replies. A conversation administrator can also delete messages.
