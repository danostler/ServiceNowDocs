# Replicating data with Instance Data Replication

Synchronize data between instances using Instance Data Replication \(IDR\).

## Overview of replicating data

IDR enables you to replicate data from one instance, called the producer instance, to one or more other instances, called consumer instances. IDR offers several replication types and options for modifying replication data to suit different needs.

-   **[Set up continuous replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/servicenow-platform/instance-data-replication-idr/set-up-continuous-replication-idr.md)**  
Enable near-real time data replication from a producer instance to one or more consumer instances in Instance Data Replication \(IDR\).
-   **[Set up bidirectional replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/servicenow-platform/instance-data-replication-idr/set-up-bidirectional-replication-idr.md)**  
Replicate updated data on a consumer instance to the producer instance using bidirectional replication in Instance Data Replication \(IDR\).
-   **[Set up discrete replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/servicenow-platform/instance-data-replication-idr/set-up-discrete-replication-idr.md)**  
Replicate data from a producer instance to specific, distinguished consumer instances in near-real time, ensuring each consumer instance receives updates independently in Instance Data Replication \(IDR\).
-   **[Set up scheduled replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/servicenow-platform/instance-data-replication-idr/set-up-scheduled-replication-idr.md)**  
Replicate data from a producer to a consumer at scheduled times each day in Instance Data Replication \(IDR\).
-   **[Seed a consumer instance](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/servicenow-platform/instance-data-replication-idr/seed-consumer-instance.md)**  
Seed all or some of the tables in your Instance Data Replication \(IDR\) replication sets depending on the needs of your consumer instance.
-   **[Modifying replication behavior in Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/servicenow-platform/instance-data-replication-idr/modifying-replication-behavior-idr.md)**  
Modify how data is replicated by configuring optional settings in Instance Data Replication \(IDR\).
-   **[Transform replication data](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/servicenow-platform/instance-data-replication-idr/modify-replicated-data.md)**  
Transform producer instance data for tables or table columns that are named differently on consumer instances in Instance Data Replication \(IDR\).
-   **[Reconfigure a replication set in Instance Data Replication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/servicenow-platform/instance-data-replication-idr/change-replication-set-config.md)**  
If you must change or update your original replication set, reconfigure Instance Data Replication \(IDR\) replication sets on the producer instance.
