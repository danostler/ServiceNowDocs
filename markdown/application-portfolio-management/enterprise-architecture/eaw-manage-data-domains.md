# Manage data domains

A data domain is a logical grouping that categorizes information objects \(like tables, fields, or records\) based on shared characteristics, business functions, or governance needs.

-   **[View all data domains](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-portfolio-management/enterprise-architecture/eaw-view-all-data-domains.md)**  
You can view the list of all data domains in the Enterprise Architecture Workspace.
-   **[Add or edit a data domain from the Portfolio page](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-portfolio-management/enterprise-architecture/eaw-add-or-edit-a-data-domain-from-the-portfolio-page.md)**  
Create or edit a data domain to relate an information object to the database catalog of a database instance to collect the physical data.
