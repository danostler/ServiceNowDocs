# Create or edit an artifact category - Legacy

Create  or edit an artifact category. Assign the category to an architectural artifact. Categories enable you to categorize and manage artifacts more efficiently.

## Before you begin

**Important:**

Starting with the Xanadu release, the legacy architectural artifact module is moved to the Enterprise Architecture Workspace. To learn more, see [Add or edit an architectural artifact category](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-portfolio-management/eaw-add-edit-arch-art-category.md).

Role required: sn\_apm.apm\_user

## Procedure

1.  Navigate to **All** &gt; **Enterprise Architecture** &gt; ** Architectural Artifacts** &gt; **Categories**.

2.  Create or edit an architectural category.

    -   To create a new architectural category, click  **New**.
    -   To modify an existing category, click the name of the category.
3.  On the form, fill in the fields.

    For field information, see [Architectural category form](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/application-portfolio-management/architectural-category-form.md).

4.  Click **Submit** or **Update**.
