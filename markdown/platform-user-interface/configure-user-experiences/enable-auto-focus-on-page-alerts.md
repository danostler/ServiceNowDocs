# Enable auto-focus on page alerts preference

Learn more about the Enable auto-focus on page alerts accessibility preference.

\[Omitted image "pol-pref-auto-focus.png"\] Alt text: Enable auto-focus on page alerts accessibility preference.

The purpose of this preference is to ensure alerts are immediately accessible by automatically moving keyboard focus to them when they appear.

<table id="table_td5_phh_2gc"><thead><tr><th>

Detail

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Feature

</td><td>

Automatically shifts focus to any page alert as soon as it appears.

</td></tr><tr><td>

Why this feature matters

</td><td>

Page alerts often contain critical information such as errors, warnings, or confirmations that users need to act on immediately. Without auto-focus, these alerts can be missed, especially when relying on screen readers or keyboard navigation. Auto-focus ensures alerts are brought to attention instantly, reducing the risk of missed notifications.

</td></tr><tr><td>

Who benefits

</td><td>

-   Keyboard-only users: focus moves directly to the alert, eliminating multiple tab steps.
-   Screen reader users: alerts become immediately discoverable without extra navigation.
-   Anyone looking to improve speed and reduce errors by prioritizing critical alerts.

</td></tr></tbody>
</table>
