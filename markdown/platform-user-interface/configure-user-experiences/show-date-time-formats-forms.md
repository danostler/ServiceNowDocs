# Show date and time formats on forms preference

Learn more about the Show date and time formats on forms accessibility preference.

\[Omitted image "coral-pref-date-time.png"\] Alt text: Show date and time formats on forms accessibility preference.

The purpose of this preference is to provide clear and consistent date and time information.

<table id="table_td5_phh_2gc"><thead><tr><th>

Detail

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Feature

</td><td>

Clearly displays the expected date and time input format to help prevent errors and confusion.

</td></tr><tr><td>

Why this feature matters

</td><td>

When forms explicitly show the date and time format \(like MM/DD/YYYY or DD/MM/YYYY\), users see exactly how to enter the information correctly. This preference is especially helpful in global contexts where formats differ by region, reducing mistakes and improving form completion rates.

</td></tr><tr><td>

Who benefits

</td><td>

-   Users in different locales with varying date/time formats
-   Screen reader users needing clear input guidance
-   Anyone filling out forms quickly

</td></tr></tbody>
</table><table id="table_qym_c3h_ngc"><thead><tr><th>

Activated

</th><th>

Deactivated

</th></tr></thead><tbody><tr><td>

Date and time fields display the format in parentheses. For example, Date \(YYYY-MM-DD\).

\[Omitted image "a11y-pref-date-time-off.png"\] Alt text: Due date form field with Show date and time formats on forms preference turned on.

</td><td>

Date and time fields do not display the format.

\[Omitted image "a11y-pref-date-time-on.png"\] Alt text: Due date form field with Show date and time formats on forms preference turned off.

</td></tr></tbody>
</table>
