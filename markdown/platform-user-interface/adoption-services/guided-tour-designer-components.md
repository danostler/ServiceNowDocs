# GTD components

Learn about the Guided Tours components.

Guided Tours are composed of the following components that function as building blocks for the tours that you create.

<table id="table_nml_pcz_pyb"><thead><tr><th>

Component

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Introduction

</td><td>

The introduction outlines the steps in the guided tour and describes the milestone you achieve when all actions are completed.

</td></tr><tr><td>

Step

</td><td>

A discrete component of a guided tour that indicates a sequence.

 A step can provide a definition, an instruction, or both.

</td></tr><tr><td>

Callout

</td><td>

The callout provides the definition and instructions for the current tour step, including guidance to move to the next step. It highlights the relevant element and provides step-by-step instructions for navigating the ServiceNow® user interface.

 You can position callouts anywhere in your user interface to provide additional context or task-related instructions. There are several display options for the callouts, including:

-   Above
-   Below
-   To the left
-   To the right

 You can customize the Service Portal guided tour callout colors. For more information, see [Guided Tour callout color](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/platform-user-interface/adoption-services/guided-tour-callout-color.md).

</td></tr><tr><td>

Element

</td><td>

The specific components that the tour refers to, such as database records or page components.

 Elements require formatting so that the tour can link to them.

</td></tr><tr><td>

Trigger

</td><td>

Triggers are actions that you take to advance the tour to the next step. Triggers include pressing the Enter key, selecting or right-clicking a UI element, or selecting a **Next** button in the callout.

 For more information, see [GTD triggers](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/platform-user-interface/adoption-services/guided-tour-triggers.md).

</td></tr><tr><td>

Conclusion

</td><td>

A conclusion indicating that the tour is complete. It is a great opportunity to encourage users to explore other tours or provide follow-up information on what to do next.

</td></tr></tbody>
</table>
