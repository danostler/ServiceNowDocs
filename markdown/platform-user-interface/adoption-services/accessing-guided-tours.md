# Start a guided tour

Guided Tours are available across four user interface types: Standard, Service Portal, Workspace, and Custom UI.

The way-guided tours start varies depending on the specific interface type \(tour type\).

**Parent Topic:**[Using Guided Tours](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/platform-user-interface/adoption-services/using-guided-tours.md)

## Start guided tour from Help Center

If you have the appropriate roles, you can access a guided tour from Standard Forms and Lists UI, Workspace, and Custom UI.

### Before you begin

Role required: user

### Procedure

1.  Select the \[Omitted image "gtd-bluehelp-icon.png"\] Alt text: Help icon to open the Help Center.

    The Help Center lists a maximum of four tours.

2.  To view more tours, select **Show more**.

3.  Select a tour from the list.

    After you begin a tour, you can see its progress in the Help Center.

    You can select **End Tour** to end the tour.

    **Note:** This process applies to custom UI pages or applications accessed through the navigator. Tours aren't visible if a custom page is opened directly without using the navigator.


## Start a guided tour from Service Portal

If you have the appropriate roles, you can access a guided tour from Service Portal.

### Before you begin

Role required: user

### Procedure

1.  Go to the Service Portal page that has a guided tour.

    The \[Omitted image "gtd-tours-button.png"\] Alt text: Tours menu appears on the banner with a color indicator to the right of the menu.

2.  Select the **Tours** menu.

    All available tours for the page appear.

3.  Select a tour.

    **Note:**

    If a page doesn’t have a Service Portal Header Menu, you can’t access the **Tours** menu. In this case, the guided tour is only available to you through auto-launch. See [Configure auto-launch for Guided Tours](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/platform-user-interface/adoption-services/auto-launch-guided-tours.md).

    If a tour is assigned at least one role, the audience is limited to users with that role.
