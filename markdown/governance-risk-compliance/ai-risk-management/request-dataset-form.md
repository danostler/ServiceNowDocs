# Request a dataset form

The Request a dataset form is designed to streamline the request process for procuring a dataset. This intake form confirms that all necessary details, supporting documents, and compliance considerations are captured before moving forward with the approval process.

See the following table for a description of the field values.

<table id="table_l35_zwl_kkb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td class="sub-head" colspan="2">

Details

</td></tr><tr><td>

Name

</td><td>

Unique name for the dataset. For example, `Employees' financial data`.

</td></tr><tr><td>

State

</td><td>

Indicates whether the dataset is in a draft, development, or deployed state.

</td></tr><tr><td>

Version

</td><td>

Version number for the dataset. For example, v1.0.

</td></tr><tr><td>

Data type

</td><td>

Data type to classify dataset. The following options are available.

-   **Image**
-   **Table**
-   **Text**
-   **Video**

</td></tr><tr><td>

Description

</td><td>

Brief description of the dataset, its core functionality, and intended use. For example, `The Employee Financial Data dataset contains comprehensive information about employees' financial status, including salary details, bonuses, benefits, and deductions. It may also encompass information on expenses, savings, and investment contributions, providing a holistic view of employees' financial profiles for training AI models.`

</td></tr><tr><td class="sub-head" colspan="2">

Ownership

</td></tr><tr><td>

Provider

</td><td>

Organization responsible for providing the dataset. For example, ServiceNow.

</td></tr><tr><td>

Managed by

</td><td>

User responsible for managing the dataset.

</td></tr><tr><td class="sub-head" colspan="2">

Additional Details

</td></tr><tr><td>

Source

</td><td>

Relevant information about the source of the dataset. For example, `The employee financial data is sourced from our internal financial application, which securely manages and processes all payroll, benefits, and compensation information. This system ensures accurate and up-to-date records for each employee's financial details.`

</td></tr><tr><td>

Base datasets

</td><td>

Foundational or primary dataset used as a reference for analysis, modeling, or further data processing.

</td></tr><tr><td>

Dataset card

</td><td>

Detailed documentation on dataset's purpose, architecture, performance, and ethical considerations for transparency. For example, this dataset is limited to loan applications from regions 25–35 and may reflect bias due to demographic constraints.
