# Set up the Operational vulnerability type

Set up the Operational vulnerability type. By defining the type and nature of the Operational vulnerability, Operational Resilience administrators can efficiently assign the operational vulnerabilities to the appropriate teams and address any issues in a timely manner.

## Before you begin

Role required: sn\_oper\_res.admin

## About this task

Administrators of Operational Resilience can create only one Vulnerability Type record within the Operational Resilience Workspace.

## Procedure

1.  Navigate to **All** &gt; **Operational Vulnerability** &gt; **Vulnerability Types**.

2.  On the Vulnerability Type record form, fill in the fields.

    The Vulnerability Type record form is shown in the example.

    \[Omitted image "vul-type.png"\] Alt text: Vulnerability Type record.\[Omitted image "vul-type-asmt-config.png"\] Alt text: Assessment.\[Omitted image "vul-type-temp-config.png"\] Alt text: Template.

    For more information on the fields, see [Vulnerability Type form](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/governance-risk-compliance/vul-type-form.md).

3.  Select the Smart Assessment template for the vulnerability type.

4.  To update the record, select **Update**.

    The Smart Assessment template is used to process the operational vulnerability.

5.  To delete a record, select **Delete**.
