# Roles installed with Digital resilience incident reporting

Certain roles are installed with the Digital resilience incident reporting functionality.

## Roles used with Digital resilience incident reporting

The following roles are used for reporting incidents with Digital resilience incident reporting.

<table id="table_shh_mnh_ydc"><thead><tr><th>

Role

</th><th>

Description

</th></tr></thead><tbody><tr><td>

sn\_dri\_inc\_rptg.digital\_resilience\_incident\_admin

</td><td>

Role for setting up administrative and Digital resilience incident activities.

 Users with this role can create and delete a few Operational Resilience activities and Digital resilience incident activities. A user with this role has the following permissions:

-   Create access to all cases
-   Read access to all cases
-   Write access to all cases

</td></tr><tr><td>

sn\_dri\_inc\_rptg.digital\_resilience\_incident\_manager

</td><td>

Role for creating Operational Resilience and Digital resilience incident activities.

 Users with this role can create and delete a few Operational Resilience activities and Digital resilience incident activities. A user with this role has the following permissions:

-   Create access to all cases
-   Read access to all cases
-   Write access to the DRI case where the manager is assigned as an analyst

</td></tr><tr><td>

sn\_dri\_inc\_rptg.digital\_resilience\_incident\_user

</td><td>

Role for participating in Operational Resilience and Digital resilience incident activities.

 Users with this role can create and delete a few Operational Resilience activities and Digital resilience incident activities. A user with this role has the following permissions:

-   Create access to all cases
-   Read access to all cases
-   Smart assessment actor access

</td></tr></tbody>
</table>
