# Create an observation for an audit task

Create an observation for an audit task to comply with federal state and local security legal requirements by studying the existing and new security legislation.

## Before you begin

Role required: sn\_audit\_ws.auditor, sn\_audit.user

## About this task

Creating an observation also helps enforce adherence to requirements, thereby advising the management on the needed actions to adhere to the federal and local security legal requirements.

## Procedure

1.  Navigate to **All** &gt; **Audit** &gt; **Audit Workspace**.

2.  Click the tasks icon \(\[Omitted image "TasksIcon.jpg"\] Alt text: Tasks icon.\).

    The workspace opens listing **All** tasks in the **My pending tasks** tab. You can view **All – Audit tasks** assigned to you.

3.  Click the audit task record for which you want to create an observation in the **Name** column.

    The record opens in the **Overview** tab displaying the workflow of its state.

4.  Click the **Observations** tab.

5.  Click **New**.

6.  Enter the details in the [Create New Observation](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/governance-risk-compliance/audit-management/create-observation-audit-ws.md) form.

7.  Click **Save**.
