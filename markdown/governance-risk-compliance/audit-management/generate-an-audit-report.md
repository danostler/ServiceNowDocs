# Generate an audit report from an engagement

Generate an audit report and maintain different versions of audit reports from an engagement in Follow up state.

## Before you begin

Role required: sn\_audit.manager

## About this task

When an engagement is in the **Follow up** state, the audit lead can generate an audit report. An audit report is represented in the form of a knowledge base \(KB\) article. After an audit report is generated, an incremented version of the report is created instead of a new report or KB article. After you make major changes to an audit report and publish the report, a new major version of the report is created.

## Procedure

1.  Navigate to **All** &gt; **Audit** &gt; **Engagements** &gt; **All Engagements**.

2.  Open the engagement for which you want to generate an audit report.

3.  Click the **Report** tab.

    1.  In the **Knowledge base** field, select the knowledge base.

    2.  In the **Report template** field, select the template.

4.  Click **Generate report**.

5.  To publish the KB article without making any changes, click **Publish**

6.  To edit the KB article, click **Edit.**.

    The KB article page appears with the state of the KB article as **Draft** and a minor version of the article is created.

7.  Edit the KB article as necessary and click **Publish**.


## Result

The incremented version of the report appears in the **KB article** field in the **Report** tab.
