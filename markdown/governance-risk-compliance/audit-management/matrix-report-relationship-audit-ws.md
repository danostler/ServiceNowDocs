# Configure target table fields as columns for matrix report

Configure the fields in the target table as column heads in the matrix report, which is the second level of configuration for the audit matrix report.

## Before you begin

Role required: sn\_matrix\_report.admin

## Procedure

1.  Navigate to **All** &gt; **Matrix report** &gt; **Matrix report registry**.

2.  Select the Matrix report registry **Name** that you created.

3.  To configure the columns in the matrix report, select the record in the **Name** column of the Matrix relationships related list.

4.  Select **New** in the Matrix column configurations related list to configure the field in the target table.

5.  On the form, fill in the fields.

    For a description of the field values, see [Matrix column configuration form](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/governance-risk-compliance/audit-management/matrix-col-config-audit-ws.md).

6.  Select **Submit**.

    You have configured one field from the target table as a column in the matrix report. You can configure other fields as well in the target table as columns for the matrix report by selecting **New** again in the Matrix column configurations related list.


## What to do next

After the matrix relationship is complete, you can [Create matrix report configuration for Audit Workspace](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/governance-risk-compliance/audit-management/matrix-report-create-audit-ws.md).
