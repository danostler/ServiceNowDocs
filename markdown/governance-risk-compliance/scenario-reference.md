# Scenario New record form

Use the Scenario New record form to create a scenario and test how an event that is associated with a scenario can impact your organization.

## Scenario New record form

For a description of the field values, see the following table.

Field

Description

</th></tr></thead><tbody><tr><td>

Name of the scenario.

</td></tr><tr><td>

Description of the scenario.

</td></tr><tr><td>

Number

</td><td>

Number of the scenario record. This field is auto-filled.

</td></tr><tr><td>

Active

</td><td>

Option to mark the scenario as active.

</td></tr><tr><td>

Pillars

</td><td>

List of pillars that you can associate with the scenario. The following pillars are provided with the Operational Resilience base system: -   Facilities
-   People
-   Process
-   Suppliers
-   Technology

</td></tr><tr><td class="sub-head" colspan="2">

Related list \(Displayed after a scenario is added\)

</td></tr><tr><td>

Events

</td><td>

Events that are related to a scenario. The related list displays the name of the event, its active status, description, associated event group, and its order that is related to the scenario. The order for the event is tied to the scenario and it is auto-filled by default in the related list.**Note:** You can add a new event or edit an existing event that is related to a scenario.
