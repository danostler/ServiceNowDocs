# Create New ICT service supply chain form

On the Create New ICT service supply chain form, fill in the fields.

<table id="table_nt2_kj4_ycc"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Number

</td><td>

Unique number of the supply chain. For example, SCHAIN0001001. This field is auto-filled.

</td></tr><tr><td>

Contractual arrangement reference number

</td><td>

Contractual arrangement for the supply chain.

</td></tr><tr><td>

Recipient of sub-contracted ICT services

</td><td>

Recipient of sub-contracted ICT services of the supply chain. It is the parent supplier.

</td></tr><tr><td>

Service provider type

</td><td>

Option to select if the supplier is a third-party or third-party engagement. Available options are:-   **Third party**
-   **Third-party engagement**
-   **Legal entity**

If you select **Third party**, the **Third party providing the service** field is displayed.

If you select **Third-party engagement**, the **Third-party engagement providing the service** field is displayed.

</td></tr><tr><td>

Identification code of the recipient of sub-contracted ICT services

</td><td>

Identification code of the recipient of sub-contracted ICT services of the supply chain. This field is auto-filled.

 Identification code of the recipient of sub-contracted ICT services of the supply chain. This field is auto-filled from the identification code of the linked ICT third-party service provider \(or legal entity\) and is kept in sync automatically. If the linked provider's identification code is updated, this field is updated in all associated supply chain record \(across all ranks\) to reflect the current code value.

</td></tr><tr><td>

Identification code of the ICT third-party service provider

</td><td>

Code to identify the ICT third-party service provider of the supply chain.

</td></tr><tr><td>

Type of code to identify the recipient of sub-contracted ICT services

</td><td>

Type of code to identify the recipient of sub-contracted ICT services of the supply chain. This field is auto-filled.

</td></tr><tr><td>

Type of code to identify the ICT third-party service provider

</td><td>

Type of code to identify the ICT third-party service provider of the supply chain. This field is auto-filled.

</td></tr><tr><td>

Type of ICT services

</td><td>

Category of the ICT services. You can choose one of the following options:-   **ICT project management**
-   **ICT Development**
-   **ICT help desk and first level support**
-   **ICT security management services**
-   **Provision of data**
-   **Data analysis**
-   **ICT, facilities and hosting services \(excluding Cloud services\)**
-   **Computation**
-   **Non-Cloud Data storage**
-   **Telecom carrier**
-   **Network infrastructure**
-   **Hardware and physical devices**
-   **Software licensing \(excluding SaaS\)**
-   **ICT operation management \(including maintenance\)**
-   **ICT Consulting**
-   **ICT Risk management**
-   **Cloud services: IaaS**
-   **Cloud services: PaaS**
-   **Cloud services: SaaS**

</td></tr><tr><td>

Rank

</td><td>

Rank of the supply chain.

 The 'rank' of an ICT third-party service provider is determined by its position in the ICT service supply chain. A direct ICT third-party service provider signing a contract with the financial entity is considered 'rank' 1. Subcontractors signing with a direct provider or another subcontractor are assigned subsequent ranks \(2, 3, etc.\). If multiple providers are at the same level, they are assigned the same 'rank'.

</td></tr></tbody>
