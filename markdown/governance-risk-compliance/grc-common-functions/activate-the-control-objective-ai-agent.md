# Activate the control objective change agent

Enable the control objective change agent in AI Agent Studio to automate compliance workflows using generative AI.

## Before you begin

Role required: sn\_grc\_sharegenai.compliance\_library\_aiagent\_user

Install the Now Assist for Integrated Risk Management \(IRM\) plugin \(sn\_irm\_gen\_ai\).

## Procedure

1.  Navigate to **All** &gt; **AI Agent Studio** &gt; **Create and manage**.

2.  Select **AI agents** from the Ready-made agentic workflows and AI agents section.

3.  From the list of available AI agents, select **Control Objective Change Agent**.

4.  On the Control objective change agent page, perform the following steps:

    1.  In the Define the specialty section, review the details and select **Continue** to proceed to the next section.

    2.  In the **Add tools and information** section, review the details and select **Continue** to proceed to the next section.

    3.  In the **Define security controls** section, review the details in the **Define user access** and **Define data access** subsections, and select **Continue**.

    4.  In the **Add triggers** section, review the details and select **Continue** to proceed to the next section.

    5.  In the **Select channels and status** section, in the **Activation status** section, turn on the **This AI agent is active** toggle switch to make the AI agent available in related workflows.

    6.  Select **Save and test**.

    7.  On the Test AI reasoning page, select the **Version** and then enter the details for **Task**.

    8.  Initiate the testing of the AI agent by selecting **Start test**.
