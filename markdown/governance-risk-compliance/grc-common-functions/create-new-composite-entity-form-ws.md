# Create new composite entity form

Use the Create New Entity form to create a composite entity by combining two or more entities from different entity classes using the Composite Entity Management application.

See the following table for a description of the field values.

<table id="table_l35_zwl_kkb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Structure

</td><td>

Structure of entity. The options are as follows:-   **Single**: Option to create a single dimension entity.
-   **Composite**: Option to create a multidimensional entity.

**Note:** Select this option as you’re creating a composite entity.


</td></tr><tr><td>

Active

</td><td>

Status of the entity.

</td></tr><tr><td>

Class

</td><td>

Option to select the composite entity class for which you want to create a composite entity. For more information on how to create a composite entity class, see [Create an entity class](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/governance-risk-compliance/grc-common-functions/create-new-entity-class-ws.md).

</td></tr><tr><td>

Owner

</td><td>

Owner of the composite entity.

</td></tr><tr><td>

Applicable entities

</td><td>

Option to select the entities for each entity class. Select the Add \[Omitted image "plus-icon.png"\] Alt text: Add icon icon. When you search for an entity, the application displays the entity hierarchy, and you can also view the complete content tree.

</td></tr><tr><td>

Name

</td><td>

Name of the composite entity. The name is automatically generated based on the selected entity classes, with a separator between them. To change the separator, see [Governance, Risk, and Compliance properties](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/governance-risk-compliance/grc-common-functions/grc-properties.md).

</td></tr><tr><td>

Description

</td><td>

Information about the composite entity.

</td></tr><tr><td>

Target

</td><td>

Option to mark the entity as one that participates in ESG target tracking. When selected, the entity can be linked to operational sustainability targets, such as measurable progress milestones toward goals like emissions reduction.

</td></tr><tr><td>

Framework

</td><td>

ESG reporting framework associated with the entity, such as GRI or SASB. This field appears only when the Target check box is selected. Selecting a framework scopes the entity's target and metric reporting to that disclosure standard.

</td></tr><tr><td class="sub-head" colspan="2">

</td></tr><tr><td>

Additional comments

</td><td>

Additional information about the composite entity.

Settings

</td></tr><tr><td>

Functional domain of the composite entity.
