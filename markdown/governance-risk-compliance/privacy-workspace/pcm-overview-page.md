# Privacy Case overview page

The Privacy case overview page shows details, such as description, state, schedule and milestones, case tasks, and issues related to the case in the Privacy Case Management application.

The Privacy Case Management overview related list includes the following sections:

-   Description
-   State
-   Case schedules and milestones
-   Case tasks overview
-   Tracking
-   Attachments

## Description section

The Description section includes the detailed information that you can fill in when you’re creating a privacy case. For example, you can add the purpose, action being taken, time line, applicable rules and regulations, and any relevant evidence or documents. By using this section, you can understand the context of the case and take the necessary steps to resolve it.

## State section

The State section includes the workflow states of a privacy case. For example in this section, you see various stages of the privacy case, such as New, Triage, Investigate, Resolve, Post case review, and Close. These states determine the status of the case. A privacy case administrator can configure these states in the Privacy Case Management application. For more information, see [Define the workflow states for a privacy case](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/governance-risk-compliance/privacy-workspace/define-pcm-workflow-states.md).

## Case schedules and milestones section

The Case schedules and milestones section includes the details on all the milestones or events during the case life cycle in a time-line range. In this section, you can see an outline of the key events, such as the date when the case was reported, occurred, discovered, or closed. You can also filter the events to view the following options on the time-line range after you select the **Show events** option:

-   **Case creation date**

    Date when the case was created.

-   **Date of discovery**

    Date when the case was discovered.

-   **Date of occurrence**

    Date when the case occurred.

-   **Milestone**

    Dates related to the investigation remediation milestones.

-   **Breach**

    Dates related to the breach analysis.

-   **Today**

    Current date on the time line.

-   **Case closure**

    Date when the case was closed or met the case closure service level agreement \(SLA\).

-   **Case duration**

    Date when the case was created to the last milestone date.

## Case tasks overview section

The case task overview section includes the following subsections:

-   **Tasks**

    Total number of case tasks, any overdue case tasks, and case tasks that are due in seven days.

-   **Ongoing tasks by state**

    Number of active case tasks by the states.

-   **Tasks by type**

    Number of active case tasks by the case type.


## Tracking section

The Tracking section displays the following information:

-   Number of open issues for a privacy case
-   Number of overdue issues
-   Number of issues due in the next seven days

## Attachments section

The Attachment section enables you to upload a file as an attachment to a case. In this section you can find all the attachments that have been added.
