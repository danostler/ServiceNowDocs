# Create a PI data element type

Create categories to help responders to identify personal data elements within specific categories. For example, for a PI data element type such as Personal information, the PI data elements can be Name, Age, Employer ID, Marital Status, Email, and so on.

## Before you begin

Role required: sn\_privacy\_case.privacy\_case\_admin

## Procedure

1.  Navigate to **All** &gt; **Privacy Case Management** &gt; **Breach Assessment Configuration** &gt; **PI Data Element Types**.

2.  Select **New**.

3.  On the form, fill in the fields.

    |Field|Description|
    |-----|-----------|
    |Label|Category to which the PI data element type belongs. For example, for type `Financial Information`, the data elements can be bank account numbers, credit card numbers, and so on.|
    |Name|Name of the PI data element type. This name is usually the same as the label.|
    |Order|Order of the element type as it should appear on the assessment form.|
    |Application|Name of the application this element type applies to. This field is automatically set to **GRC: Privacy Case Management**.|
    |Active|Option to activate the data element type.|
    |Description|Brief description of the PI data element type.|

4.  Select **Submit**.
