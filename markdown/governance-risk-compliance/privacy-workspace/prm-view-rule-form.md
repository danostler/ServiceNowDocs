# View rule form

Learn about the fields on the view rule form. Use this form to define workspace and default view rules in the Privacy Case Management application.

On the view rule form, fill in the fields.

<table id="table_FloorForm"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Name

</td><td>

Unique name to identify the rule.

</td></tr><tr><td>

Table

</td><td>

Table this view rule applies to. This field is automatically set to **Privacy case \[sn\_privacy\_case\_privacy\_case\]**.

</td></tr><tr><td>

View

</td><td>

View for which you’re creating the rule. The choices are as follows:-   **Default view**: Select this option to define the form view on the Core UI.
-   **Workspace**: Select this option to define the form view on the Privacy Workspace.

</td></tr><tr><td>

Experience Restricted

</td><td>

Option to decide if the rule must be restricted to one workspace or to multiple workspaces.

</td></tr><tr><td>

Application

</td><td>

Name of the application this view rule applies to. This field is automatically set to **GRC: Advanced Risk**.

</td></tr><tr><td>

Execution Order

</td><td>

Order in which this view rule executes if there are multiple rules.

</td></tr><tr><td>

Active

</td><td>

Option to activate the view rule.

</td></tr></tbody>
