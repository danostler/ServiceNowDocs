# Personal data rights request form

Use the Personal data rights request form in the Personal Data Rights application to initiate a process to access, correct, or delete personal data.

The following table lists the descriptions of the field values.

<table id="table_lrv_yw4_brb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Number

</td><td>

Request number. This field is automatically set.

</td></tr><tr><td>

Source

</td><td>

Source from which the request is initiated. The choices are as follows. -   **Email**
-   **Phone**
-   **Employee Center**
-   **Chat bot**
-   **Public API**

</td></tr><tr><td>

Priority

</td><td>

Priority of the request. -   **1 - Critical**
-   **2- High**
-   **3 - Moderate**
-   **4 - Low**
-   **5 - Planning**

</td></tr><tr><td>

State

</td><td>

Workflow state of the request. This field is automatically set to **New**.

</td></tr><tr><td>

Name

</td><td>

Name of the request. For example, `Rectification of data`.

</td></tr><tr><td>

Description

</td><td>

Brief description of the request. For example, `An employee discovered that their address is incorrectly recorded in our accounts. The employee called requesting access to their personal data and specifically ask for the correction of the address. As per the process, respective Data Admin should call the employee and get the new address address details to correct in the HR systems.`

</td></tr><tr><td>

Regulation

</td><td>

Regulation that governs the request.

</td></tr><tr><td class="sub-head" colspan="2">

Requester details

</td></tr><tr><td>

Requester name

</td><td>

Name of the person who requests the data.

</td></tr><tr><td>

Location

</td><td>

Requester's geographical location.

</td></tr><tr><td>

Email

</td><td>

Email ID of the requester.

</td></tr><tr><td>

User validation

</td><td>

Method used to validate and confirm the user. The choices are as follows. -   **IVR \(Interactive voice response\)**
-   **To be validated**
-   **Security questions \(Manual verification\)**
-   **SSO \(Single sign on\)**

</td></tr><tr><td>

Requester type

</td><td>

Category of the requester. The choices are as follows. -   **Prospect**
-   **Customer**
-   **Employee**
-   **Ex-Employee**

</td></tr><tr><td>

Request type

</td><td>

Type of request made by the requester. The choices are as follows:-   **Right to correct**
-   **Right to delete**
-   **Right to know**
-   **Right to opt-out**

</td></tr><tr><td>

Phone number

</td><td>

Phone number of the requester.

</td></tr><tr><td>

Business unit

</td><td>

Business unit of the requester.

</td></tr><tr><td>

Validation comments

</td><td>

Additional comments to validate the user.

Due date for the completion of the request.

</td><td>

Start date of request processing.

Assignment

</td></tr><tr><td>

Verification approvers
