# Create connections and authenticate their credentials

Establish a connection and authenticate your login credentials with an Everbridge instance. You can then send out a notification using the delivery channel. This connection helps you to send and receive communications with an Everbridge instance.

## Before you begin

Role required: sn\_bcm.admin

## Procedure

1.  Navigate to **Business Continuity** &gt; **Everbridge Notification Configuration** &gt; **Connection &amp; Credential Alias**.

2.  Select the **Create New Connection &amp; Credential** related link.

3.  On the form, fill in the fields.

    For more information on the fields, see [Create Connection and Credential form](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/governance-risk-compliance/create-connection-and-credential-form.md).

4.  Select **Create**.

    A connection is established to send and receive communications with an Everbridge instance.
