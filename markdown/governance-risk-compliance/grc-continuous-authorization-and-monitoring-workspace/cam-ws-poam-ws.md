# Managing POA&amp;Ms issues

In CAM application, all issues related to an authorization package are called as Plan of Actions and Milestones \(POA&amp;Ms\). The issues can be package issues, or control issues, engagement issues, or control test issues that are related to the package.

In the POA&amp;Ms page you can view all the authorization package issues in different states that belong to the CAM application.

\[Omitted image "cam-ws-poams-page.png"\] Alt text: Plan of actions and milestones or issues landing page.

The authorization package filter, exclusive to the POA&amp;Ms page, filters all the POA&amp;Ms associated with the selected packages.

**Note:** The Issue to Authorization Package \[sn\_irm\_cont\_auth\_m2m\_issue\_auth\_pack\] table stores the data related to POA&amp;Ms associated with with authorization packages.

-   **Overview**

    POA&amp;Ms: The donut charts display the total number of POA&amp;Ms, by default grouped by state. However, POA&amp;Ms can also be grouped by priority, POA&amp;Ms type, and rating. The visualization displays only the active POA&amp;Ms. You can apply the filter to visualize the data however you want.

    Overdue POA&amp;Ms: The stacked bar charts display the overdue POA&amp;Ms grouped by due date and stacked by priority. The stacking can also be updated to POA&amp;Ms rating.

    Observation by state: The donut chart displays POA&amp;Ms in different states.

    Observations aging by priority: The stacked bar displays POA&amp;Ms by priority.

-   **Performance**

    Displays the open and closed POA&amp;Ms trend.

-   **My POA&amp;Ms**

    Monitor the POA&amp;Ms assigned to you, which are in different states.

-   **My group's POA&amp;Ms**

    Monitor the POA&amp;Ms assigned to the user groups that you belong to.


## Required roles

All users who can access the CAM Workspace can also access the POA&amp;Ms landing page. For a complete list of CAM Workspace users, see [List of users who can access CAM Workspace](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/governance-risk-compliance/grc-continuous-authorization-and-monitoring-workspace/cam-ws-home-page.md).

## Access the POA&amp;Ms landing page

To access the POA&amp;Ms page, navigate to **Workspaces** &gt; **CAM Workspace** and select the POA&amp;Ms icon \(\[Omitted image "cam-poams-icon.png"\] Alt text: POA&amp;Ms icon.\).
