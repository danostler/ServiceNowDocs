# Request an approval for the action task

Complete the work on the assessment-type or the investigation-type action task and then request an approval from the approvers of the Operational vulnerability record.

## Before you begin

Role required: sn\_oper\_res.manager, Action task owner

## Procedure

1.  Navigate to **Workspaces** &gt; **Operational Resilience Workspace** &gt; **All Operational Vulnerabilities**.

2.  Review the email notification and open the action task record from the email.

3.  Review the action task details and provide observations in the Summary section of the action task form.

4.  Select **Save**.

    The action task record is saved. The vulnerability record analyst reviews the notes and assigns the action task again to the task owner. An email notification is sent to the user.

5.  Open the action task record from the email and select **Accept work** in the action task record.

    The user accepts the work and completes the following steps for the action task of the vulnerability record.

6.  Finish work on the action task and select the **Ready for review** UI action.

    The action task is in the **Ready for review** state.

7.  Add work notes in the Confirmation window and select **Ready for review**.

    A notification is sent to the Operational vulnerability record owner or analyst about completion of the action task.


## What to do next

Devise a course of treatment plan and perform root cause analysis. For more information, see [Decide the treatment and perform a root cause analysis](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/governance-risk-compliance/deciding-treatment-analysis.md).
