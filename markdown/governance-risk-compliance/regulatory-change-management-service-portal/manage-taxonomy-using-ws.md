# Manage the taxonomy

Manage the taxonomy when you are setting up the Regulatory Change Management application in the Compliance Workspace. Create an internal taxonomy and map the internal taxonomy to the external taxonomy so that the Regulatory Change Management application refers to only one taxonomy.

## Before you begin

Role required: sn\_grc\_reg\_change.admin

## Procedure

1.  Navigate to **All** &gt; **Regulatory Change Management application** &gt; **Compliance Workspace**.

    A new Compliance Management tab is displayed.

2.  In the List view, navigate to **Lists** &gt; **Taxonomy** page.

3.  Navigate to one of the following modules:

    -   **Regulatory bodies**
    -   **Content types**
    -   **Jurisdictions**
    -   **Themes**
    -   **Sectors**
    The name and parent body for the selected taxonomy category are displayed.

4.  In the Name list, select a taxonomy element.

    Details of the selected taxonomy element are displayed on a separate page with the Details tab and Internal taxonomy related list.

5.  To create a new internal taxonomy for the selected taxonomy class, select **New**.

    |Field|Description|
    |-----|-----------|
    |Name|Name of the internal taxonomy.|
    |Taxonomy class|Taxonomy class populated by default for the selected category.|
    |Parent|Parent record for which the new internal taxonomy is created.|

6.  To save the taxonomy, select **Save**.
