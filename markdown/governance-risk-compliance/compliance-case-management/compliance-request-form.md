# Compliance request form

Use the Create new compliance request form in the GRC: Compliance Case Management application to raise a compliance request.

The following table lists the descriptions of the field values.

<table id="table_uj1_q5x_nvb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Number

</td><td>

Number of the request. This field is automatically set to a request number.

</td></tr><tr><td>

Name

</td><td>

Name of the request. For example, a request for the Policy Update - Annual medical examination is mandatory for all permanent employees.

</td></tr><tr><td>

Description

</td><td>

Brief description of the request.

</td></tr><tr><td>

Type

</td><td>

Type to classify the nature of the compliance request.

</td></tr><tr><td>

State

</td><td>

Workflow state of the request. This field is automatically set to **New**.

</td></tr><tr><td>

Priority

</td><td>

Priority of the request:-   **None**
-   **1 - Critical**
-   **2- High**
-   **3 - Moderate**
-   **4 - Low**
-   **5 - Planning**

</td></tr><tr><td>

Requested by

</td><td>

Name of the person who raised the request.

</td></tr><tr><td>

Requested on behalf of

</td><td>

Name of the person that you created the request for.

</td></tr><tr><td class="sub-head" colspan="2">

Assignment

</td></tr><tr><td>

Assignment group

</td><td>

Group that is assigned to the request.**Note:** The assignment group is preconfigured to the request type during the configuration setup.

</td></tr><tr><td>

Approvers

</td><td>

Person who should approve the request.

</td></tr><tr><td>

Assigned to

</td><td>

Person who is assigned to this request.

</td></tr><tr><td>

Watch list

</td><td>

User who must be informed about the request.

</td></tr><tr><td class="sub-head" colspan="2">

Primary origin

</td></tr><tr><td>

Business unit

</td><td>

Business unit that is affected by the request. For example, Legal.

</td></tr><tr><td>

Location

</td><td>

Location where the request was raised. For example, Japan.

</td></tr><tr><td class="sub-head" colspan="2">

Schedule

</td></tr><tr><td>

Created

</td><td>

Date when the request was raised. This field is automatically set to the current date and time.

</td></tr><tr><td>

Approved on

</td><td>

Date when the request was approved.

</td></tr><tr><td class="sub-head" colspan="2">

Activity

</td></tr><tr><td>

Work notes \(Private\)

</td><td>

Notes or information about the request.

</td></tr><tr><td>

Additional comments \(Customer visible\)

</td><td>

Additional information about the request that you want to share with your customers.

</td></tr><tr><td>

Save

</td><td>

Save the details of the compliance case.
