# View the assessments generated by event-driven management rules

When an event-driven management rule runs, it auto-generates and sends questionnaires and doc requests to third parties. You can view the current state \(running, successful, or recalled\) of all runs of rules. Select a record number to view details and to view the set of assessments generated by the rule.

## Before you begin

Role required: sn\_vdr\_risk\_asmt.vendor\_risk\_manager

## Procedure

1.  On the Vendor Management Workspace, use either of the following methods to start the process:

    -   To view all DD requests and assessments for a particular rule:

        1.  Select the list icon \[Omitted image "icon-tprm-ws-list.png"\] Alt text:.
        2.  In the **Event-driven management rules** list, select any option: **Active**, **Inactive**, or **All**.
        3.  In the **Event-driven management** list, select any option: **Active**, **Inactive**, or **All**.
        4.  Select a rule.
        5.  On the **Event-driven management history** tab, select a run.
        6.  Select the appropriate tab to view the DD requests or assessments that the rule sent.
    -   To view the DD requests and assessments for a particular run of a rule:

        1.  Select the list icon \[Omitted image "icon-tprm-ws-list.png"\] Alt text:.
        2.  In the **Event-driven management** list, select either option: **Successful runs** or **Recalled runs**.
        3.  Select a run.
        4.  Select the appropriate tab to view the DD requests or assessments that the rule sent.
2.  On the legacy interface, use the following method:

    -   To view the DD requests and assessments for a particular run of a rule:
        1.  Navigate to **All** &gt; **Third-party Risk Management** &gt; **Event-driven management** and then select either option: **Successful runs**, **Recalled runs**.
        2.  Select a run.
        3.  Select the appropriate tab to view the DD requests or assessments that the rule sent.
3.  Select the **Details** tab to view the record of a run.

    Each record includes the following read-only values:
