# Using Software Asset Management classic

Use the classic Software Asset Management framework to manage your licenses, compliance, and optimization.

-   **[Create entitlements in Software Asset Management classic](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/it-asset-management/software-asset-management/track-software-rights.md)**  
Create entitlements in the Software Asset Management application classic to record your license details and allocate purchased software rights to users or devices.
-   **[Import bulk entitlements in Software Asset Management classic](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/it-asset-management/software-asset-management/view-entitlement-import-errors.md)**  
You can import multiple entitlements together using a Microsoft Excel spreadsheet in Software Asset Management classic.
-   **[Create software models in Software Asset Management classic](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/it-asset-management/software-asset-management/record-terms-software-licenses.md)**  
Create a software model to add product details that are used to connect software rights you purchased with software installations discovered on your system.
-   **[View normalization suggestions in Software Asset Management classic](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/it-asset-management/software-asset-management/view-norm-suggestions-sam.md)**  
View normalization suggestion records that are created for discovery models. You can accept or reject these suggestions.
-   **[Software license usage](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/it-asset-management/software-asset-management/sam-license-workbench.md)**  
Review reconciliation results in a simplified workbench view.
-   **[Create averages for product life cycles in Software Asset Management classic](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/it-asset-management/software-asset-management/create-lifecycle-averages-classic.md)**  
Create averages to calculate software End of Life \(EOL\) and End of Support \(EOS\) life cycles in the Software Asset Management classic application.
-   **[Add a software client access record in Software Asset Management classic](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/it-asset-management/software-asset-management/t_AddAClientAccessRec.md)**  
Add a client access record to track and manage the users or devices that are accessing a particular version of your server software using a client access license \(CAL\).
-   **[Add a custom software product in Software Asset Management classic](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/it-asset-management/software-asset-management/t_AddACustomProduct.md)**  
If a software product does not exist in the Software Library, you can add a custom product. Custom products enable you to normalize and account for homegrown software, or software that is not yet part of the Software Library.
-   **[View publisher part number \(PPN\) suggestions in Software Asset Management classic](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/it-asset-management/software-asset-management/View-customppn-classic.md)**  
View content service suggestions for your custom PPNs and DMAPS in the Software Asset Management classic application.
