# Journey designer version 2 plan configuration fields

These fields are for Journey designer version 2 plan configurations.

<table id="table_msg_jfl_bwb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

**Title**

</td><td>

The title of the plan configuration that is visible in the Employee Center.

</td></tr><tr><td>

**Audience**

</td><td>

Used to restrict content access by an audience. See, [Restrictions for Content Publishing and Content Experience](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/employee-service-management/employee-experience-foundation/ecpro-content-restriction.md)

</td></tr><tr><td>

**Visible to all owners**

</td><td>

When selected, the plan configuration is visible to all users that own any plan configuration. **Note:** Only owners are able to change or add to the plan configuration.

</td></tr><tr><td>

**Short description**

</td><td>

A short one line description of the plan configuration, for example "Onboarding journey template for sales employees."

</td></tr><tr><td>

**Description**

</td><td>

A long description of the plan configuration that owners read to get more details about the configuration.

</td></tr><tr><td>

**Active**

</td><td>

A published plan configuration is active. If **Active** is selected, it’s available to be used.

</td></tr><tr><td>

**Journey Accelerator plan type**

</td><td>

A Journey designer plan configuration must be associated with a Journey Accelerator Plan Type.

</td></tr><tr><td>

**State**

</td><td>

-   **Awaiting approval**: The plan configuration has been published by the owner, and it must be reviewed and approved by the assigned **Approvers**.
-   **Draft**: The plan configuration is under development and it must be approved to be published and move to the **Ready** state.
-   **Ready**: A published plan configuration is in the ready state.

</td></tr></tbody>
