# Journey designer Model task fields

Managers create to-do tasks that are saved as templates. The templates are stored as a Journey designer Model Task.

<table id="table_epy_jkx_5qb"><thead><tr><th align="left">

Field

</th><th align="left">

Description

</th></tr></thead><tbody><tr><td>

Task name

</td><td>

The name of the to-do as it appears in an action plan.

</td></tr><tr><td>

Owner

</td><td>

The manager that created the to-do task template.

</td></tr><tr><td>

Additional assignee list

</td><td>

Adds user to the **Additional people** field of the to-do task.

</td></tr><tr><td>

Assigned to type

</td><td>

-   Persona: One of the personas supported for action plans. \(manager, employee, mentor\)
-   User: A single user.

**Note:** The availability of other fields depends on your selection. The persona is listed in the **Assigned to source** field.

</td></tr><tr><td>

Parent plan table

</td><td>

The table that contains the to-do task template information \(Journey Accelerator Plan \[sn\_ja\_plan\]\).

</td></tr><tr><td>

Assigned to source

</td><td>

The persona to which the to-do task is assigned.

</td></tr><tr><td>

Assigned to

</td><td>

The name of a single user.

</td></tr><tr><td>

Active

</td><td>

Select to make the task available. Clear the check mark to hide the template from users.

</td></tr><tr><td>

Mandatory

</td><td>

Select to make the to-do task mandatory. Clear the check mark to make the task optional.

</td></tr><tr><td>

Journey Accelerator task type

</td><td>

-   Regular: Task without the scheduling feature.
-   Calendar: Available when Journey Accelerator is configured to work with a calendar such as Microsoft Exchange Online
-   Learning: Delivers an online course. This option is only available when Learning Posts is installed.
-   Pulse surveys: Delivers a survey to collect feedback. This option is only available when Listening Posts is installed.

</td></tr><tr><td>

Due date configuration

</td><td>

A conditional due date for the task.

</td></tr><tr><td>

Number of days/weeks

</td><td>

The days or weeks from the assigned condition for when the task is due.

</td></tr><tr><td>

Task description

</td><td>

The description of the task that is visible to the users.

</td></tr></tbody>
