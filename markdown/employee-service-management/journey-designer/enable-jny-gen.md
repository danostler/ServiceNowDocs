# Enable the Now Assist Journey generation skill

Enable the Journey generation skill in Now Assist for HRSD using the Now Assist Admin console.

## Before you begin

Role required: sn\_jny.admin, sn\_nowassist\_admin.nsa\_admin

## Procedure

1.  Navigate to **All** &gt; **Now Assist Admin** &gt; **Skills**.

2.  On the navigation panel, select **Employee**, then select **HRSD**.

3.  On the **Journey Generation for Managers** card, select **Activate skill**.

4.  Select **Define Trigger**, the first step in the guided setup.

    By default, many of the options in the setup are configured for the most common use cases. You might need to select the step in the guided setup navigation to go back and change the configurations in previous steps. You can also use the **Back** button to navigate through the steps.

5.  Using the toggles, select what actions trigger the journey generation.

6.  Select **Save and continue**.

7.  Select **Select display**, then select where you would like to display the skill.

    You can select both in-product, Now Assist panel, or both.

    -   **In-product**: When selected, Now Assist skills are displayed on forms and workspaces.
    -   **Now Assist panel**: When selected, Now Assist skills are available in the Now Assist panel. If you don't see this option, you must activate the Now Assist panel. For more information, see [Activate the Now Assist panel standard chat](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/intelligent-experiences/enable-ai-experiences/activate-now-assist-panel.md).
    Select the arrow near the toggle to select roles for those who can access the skill. You can add roles by entering the name of the role in the **User roles** field. You can remove existing roles by selecting the X icon in the role bubble. You must have at least one role specified, but you can add as many roles as you like.

8.  Select **Review and activate** to review your choices.

9.  Complete the configuration by selecting **Activate**.
