# Define user access to course within internal learning content

Define user access to define who has read access to internal learning content.

## Before you begin

Role required: admin

Create internal learning content. For more information, see [Create internal Learning content](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/employee-service-management/learning-core/create-internal-learning-content.md).

## Procedure

1.  Navigate to **All** &gt; **Learning** &gt; **Content** &gt; **Internal Content**.

2.  Open the internal learning course that you want to define user access for.

3.  On the Can Read related list , select **Edit**.

4.  Add users who should have access to the internal learning course.

5.  On the Cannot Read related list , select **Edit**.

6.  Add users who should not have access to the internal learning course.

    **Note:** When a user is present in both Can Read and Cannot Read related lists then Cannot Read takes precedence, and the user is restricted from read access to that internal learning course.
