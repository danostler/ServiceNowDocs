# Work on a contract change request

As a contract fulfiller, review the change request, update the contract document and send it back to the contract user.

## Before you begin

Role required: sn\_cm\_core.contract \_fulfiller and sn\_lg\_cnt.contract\_fulfiller

## Procedure

1.  Navigate to **All** &gt; **Legal Request** &gt; **Legal Counsel Center**.

2.  Select the List icon \(\[Omitted image "lsd-lcc-list-icon.png"\] Alt text: List icon\).

3.  Select **Legal Requests**.

4.  Go to the contract request listing.

    -   For assigned contract request - Select **Assigned to me**.
    -   For unassigned contract request - Select **Unassigned**.
5.  Select the legal request for which the change request has been submitted.

    -   When you have opened a request from the unassigned list, assign it to yourself or the group manager has to assign it to a user.
    -   When the legal request is assigned, select **Start work**.
    -   You can get the details of the change request from the Activity stream or the notification email received.
6.  Select Contracts request tab and open the request.

    You can get the details of the change request from the Activity stream or the notification email received.

7.  Select the Contract documents tab.

8.  Access and edit the contract document.

10. Get approval of the updated document from stakeholders.

    For more information, see [Initiate an ad hoc approval for a contract document revision](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/employee-service-management/legal-service-delivery/snlc-initiate-approval-cr.md).

    1.  Select **Send document**.

    2.  Select **Send document** on the confirmation screen.


## Result

The document is sent back to the user and the Contract status updates to Document ready to indicate that the contract document has been updated according to the change request.
