# Legal Matter Task form for legal investigations

Matter owners can use the Legal Matter Task form for adding task type of Documents, Recommended Actions, and Interviews as part of the legal investigation.

<table id="table_y53_qfp_kjb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Name

</td><td>

Unique name for the task.

</td></tr><tr><td>

Type

</td><td>

Type of the task.Select **Recommended Action**.

</td></tr><tr><td>

Priority

</td><td>

Level of priority to work on and complete the task. The priority of a task determines its importance for resolving the matter.

</td></tr><tr><td>

Phase

</td><td>

Phase to which the task belongs.

</td></tr><tr><td>

Matter

</td><td>

Matter to which the task belongs.

</td></tr><tr><td>

Event name

</td><td>

Name of the calendar event associated with the task. A calendar event with a due date helps you to track the progress of the task for specific milestones.

</td></tr><tr><td>

State

</td><td>

Current state of the task to determine the progress of the task.

</td></tr><tr><td>

Order

</td><td>

Order in which the task should be worked on when there are multiple tasks in a phase or matter.

</td></tr><tr><td>

Assignment group

</td><td>

User group to which the task is assigned.

</td></tr><tr><td>

Assigned to

</td><td>

User to whom the task is assigned. If an assignment group is selected, the list shows members of the selected group.

</td></tr><tr><td>

Due date

</td><td>

Date to complete the task.

</td></tr><tr><td>

Internal party

</td><td>

Names of the involved parties to whom the recommended actions apply.

</td></tr><tr><td class="sub-head" colspan="2">

Recommended Action Details section

</td></tr><tr><td>

Recommended action type

</td><td>

Type of the recommended action such as verbal warning or suspension.

</td></tr><tr><td>

Discussion date and time

</td><td>

Date when the recommended actions should be discussed with the associated involved parties.

</td></tr><tr><td>

Additional attendees

</td><td>

Users other than the involved parties who would be part of the discussion meeting.

</td></tr><tr><td>

Description

</td><td>

Description of the recommended actions.

</td></tr><tr><td class="sub-head" colspan="2">

Notes section

</td></tr><tr><td>

Work notes

</td><td>

Comments or notes that only authorized users related to the matter can see.

</td></tr><tr><td>

Additional comments

</td><td>

Comments or notes that the requester can also see.

|Field|Description|
|-----|-----------|

The following table describes about the fields specific to the interview task type.

End date and time

Date and time when the interview ends.By default, when you select the start date and time, the end date time is automatically set to one hour after the start time. You can change based on the interview time.
