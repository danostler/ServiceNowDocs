# Configure the inclusion of sensitive data in incident patterns

Configure whether safety incident records with sensitive data should be included in the incident pattern analysis.

## Before you begin

Role required: sn\_ohs\_im.admin

## About this task

The configuration of this option in the application property decides whether incident pattern jobs running in the background should include safety incident records that contain sensitive data. Configure this setting if needed before running the incident pattern analysis.

## Procedure

1.  Navigate to **All** &gt; **Health and Safety** &gt; **Health and safety administration** &gt; **Properties**.

2.  In the **Control whether sensitive incidents are included in the pattern detection analysis** field, select

    -   Yes, to include the incident records with sensitive data in the analysis.
    -   No, to exclude incident records with sensitive date in the analysis.
3.  Select **Save**.
