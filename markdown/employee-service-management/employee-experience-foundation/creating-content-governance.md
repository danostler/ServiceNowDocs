# Creating content through Content Governance

When a content request item is assigned to you, create and schedule the content, then send it for approval.

## Before you begin

Role required: sn\_cd.content\_manager

## Procedure

1.  Navigate to **All** &gt; **Content Governance** &gt; **Assigned to me**.

2.  Click the content request you want to fulfill.

3.  Change the state field to **Work In Progress**.

4.  From the top-left menu, click **Save**.

    If you click **Update**, the system returns you to the Assigned to me list and you must reopen the content request.

5.  Scroll down to the content request item related list and select the request item.

6.  Click **Create Content**.

    Create and schedule the content in the [Portal content](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/employee-service-management/employee-experience-foundation/ec-streamline-content.md).

    **Note:** When you are creating the publish plan, the **Inactive** option is selected by default. When the content is approved, the system activates the publish plan. Alternatively, you can set a publishing duration. However, if you select **Always active**, the content will publish without approval.

7.  Click **Send for Review**.

    The system updates the content request status to `In Review`. To edit the content, you must revert the status to `In Progress`.
