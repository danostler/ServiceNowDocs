# Request filter configuration form

The field descriptions provide a reference point when you're creating a Requests filter configuration.

<table id="table_cxz_sgz_tfc"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Label

</td><td>

Name of the filter visible on the Requests page.

</td></tr><tr><td>

Request filter definition

</td><td>

Filter definition under which this filter is being configured.

</td></tr><tr><td>

Application

</td><td>

Application scope where the filter record resides.

</td></tr><tr><td>

Active

</td><td>

Option to enable the filter configuration.

</td></tr><tr><td>

Order

</td><td>

Order of priority to determine which filter configuration to prioritize.

</td></tr><tr><td>

Apply on load

</td><td>

Option to enable the filter on the Requests page even when it is reloaded.

</td></tr><tr><td>

Available for

</td><td>

Option to add the users the filter configuration is available for.

</td></tr><tr><td>

Not available for

</td><td>

Option to choose any audience if they should be excluded from accessing the filter on the Requests page.

</td></tr><tr><td>

Reference table

</td><td>

Source table for the reference filter, like location.This field appears only when **Reference** is selected from the **Filter type** in the filter definition.

</td></tr><tr><td>

Reference condition

</td><td>

Conditional suggestion like **Add Filter Condition**, **Add OR Clause**, and **Add Sort** to add to the filters.This field appears only when **Reference** is selected from the **Filter type** in the filter definition.

</td></tr><tr><td>

Search field

</td><td>

Specifics of the selected reference filter column item to be displayed on the portal page. For example, if **Location** is the filter column item, you can select **City** or **Country** on the **Search field**.

</td></tr><tr><td>

Associated Filters

</td><td>

List of filters associated with the tab.

</td></tr></tbody>
