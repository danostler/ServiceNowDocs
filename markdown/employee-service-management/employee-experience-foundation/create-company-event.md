# Create a company event

Use company events to configure a live-streamed event.

## Before you begin

-   Role required: sn\_cd.content\_manager
-   To create and publish company events with greater efficiency, use a content template. For more information, see [Create a Content template](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/employee-service-management/employee-experience-foundation/ec-create-content-template.md).
-   You can assign one or more content categories to the company event so employees can filter by category. See [Create a content category](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/employee-service-management/employee-experience-foundation/ec-content-cat.md).
-   Social engagement settings enable employees to view page count, select a reaction icon, or leave a comment. See [Content engagement](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/employee-service-management/employee-experience-foundation/ec-content-engagement.md).

## About this task

The company event content type is built using the Rich Content Editor and published to the employee portal via widgets. For an overview, see [Company events](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/employee-service-management/employee-experience-foundation/ec-company-events.md).

## Procedure

1.  Navigate to **Content Publishing** &gt; **Content Library**.

2.  Select **New**.

3.  Select **Company event** under the **Choose a content format** section.

4.  Select **Continue**.

5.  Fill in the fields in the **New content** form.

    |Field|Description|
    |-----|-----------|
    |Event content record name|The name that describes the content you are creating.|
    |Order|The order in which the event appears in a widget. An event with a lesser value appears first.|
    |Active|Indicates the content is active and available for use.|
