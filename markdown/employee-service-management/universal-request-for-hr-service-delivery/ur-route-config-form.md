# Transfer configuration form

Use the **Transfer configuration** form to configure your transfer details in the Universal Request application.

<table id="table_jb3_qpg_q5"><thead><tr><th>

Field

</th><th>

Definition

</th></tr></thead><tbody><tr><td>

Table

</td><td>

The service or department table that you want to map for.

</td></tr><tr><td>

Transfer reason

</td><td>

The reason given when the task is transferred. The options to choose are:-   **With resolution**

The primary task is addressed and is transferred with the resolution.

-   **Without resolution**

The primary task is not completed and is transferred without resolution.


</td></tr><tr><td>

Active

</td><td>

Indicates if the configuration for the department is active or not.

</td></tr><tr><td>

Mandatory fields

</td><td>

The mandatory fields that must be filled before a ticket is transferred.

</td></tr><tr><td colspan="2">

**Transfer Information**

</td></tr><tr><td>

Transfer state

</td><td>

The state of the department-specific ticket. The states are listed based on the department specific table that you select.

</td></tr><tr><td>

Transfer template

</td><td>

Select the fields that you want to populate other than the state.

</td></tr><tr><td colspan="2">

**Closure Information**

</td></tr><tr><td>

Closed state

</td><td>

State that you want to populate for the already transferred department ticket when UR is getting closed. The states are listed based on the department specific table that you select.

</td></tr><tr><td>

Closed template

</td><td>

Select the fields that you want to populate other than the state.

</td></tr></tbody>
