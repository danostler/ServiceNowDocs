# System logs

The System Logs module provides a variety of logs that you can use to troubleshoot and debug transactions and events that take place within the instance.

Access the following logs from the System Logs module:

|Log|Description|
|---|-----------|
|[Transactions](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-security/servicenow-ai-platform-security/r_TransactionLogs.md)|All application activity for an instance.|
|[Email](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-security/servicenow-ai-platform-security/r_EmailLogs.md) and [Push](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-security/servicenow-ai-platform-security/push-log.md)|All email notifications and Push messages sent from all instances within the system.|
|[Event Logs](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-administration/system-events/event-logs-2.md)|All system events that occur within the system.|
|[Import](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-security/servicenow-ai-platform-security/r_ImportLogs.md)|Data import activity within the platform.|
|Table Changes|Changes made to all tables in the system.|
|[Outbound web services logging](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/api-reference/web-services/outbound-request-logging.md)|All outbound web services requests such as REST and SOAP requests.|
|[System](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-security/servicenow-ai-platform-security/r_SystemLogs.md)|Warnings and errors for instance processes, records, and non-critical events, such as memory usage on the server machine.|

Use the [Log File Browser](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-security/servicenow-ai-platform-security/r_LogUtilities.md) to search and download logs. You can also search archived logs in the [log history](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-security/servicenow-ai-platform-security/r_LogHistory.md).

## Other logs

Your instance offers other logs in addition to those in the System Logs module. For example, the [System Diagnostics module](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-security/servicenow-ai-platform-security/c_SystemDiagnosticsApplication.md) provides upgrade history and slow query logs, which you can use to gain insight into how queries are affecting platform performance. The [Customer Updates table](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-security/servicenow-ai-platform-security/r_CustomerUpdatesTable.md) records every change that is made in the system.
