# Combined App Engine Studio release notes for upgrades from Washington DC to Zurich

Consolidated page of all release notes for App Engine Studio from Washington DC to Zurich.

## How to use this page

To help you prepare for your upgrade, we have combined the cross-family App Engine Studio release notes onto one page. Read this summary of the new features, changes, and updated information for your product from Washington DC to Zurich.

**Tip:** If there were no updates for a release notes section in a certain family release, we included a short note for your reference. For example, if a product did not have any updates in Tokyo, the row says "No updates for this release."

## Important information for upgrading App Engine Studio to Zurich

Before you upgrade to Zurich, review these pre- and post-upgrade tasks and complete the tasks as needed.

Between your current release family and Zurich, new features were introduced for App Engine Studio.

<table class="custom-rows"><thead><tr><th class="filter">

Release

</th><th>

Release notes

</th></tr></thead><tbody><tr><td>

Washington DC

</td><td>

-   **[Create applications with Now Assist](https://servicenow-staging.fluidtopics.net/access?context=now-assist-app-landing&family=washingtondc&ft:locale=en-US)**

Accelerate application development through a natural conversation with generative AI. Create tables, roles, and record producers that define your application to quickly reach a starting point for further building.

-   **[Quickly add required roles for Workspace Builder](https://servicenow-staging.fluidtopics.net/access?context=configure-workspace-settings&family=washingtondc&ft:locale=en-US)**

Quickly access and define security for workspaces that don't have an access control list \(ACL\) through a new modal. The new Add user roles to continue modal enables you to add roles and automatically create an ACL. You can then edit the roles in the **User access** field of the workspace settings.

-   **[Added support for Creator Studio groups and roles in AEMC](https://servicenow-staging.fluidtopics.net/access?context=manage-developers-using-aemc&family=washingtondc&ft:locale=en-US)**

As an admin, track and manage users who have Creator Studio permissions when you use AEMC.

-   **[Added support for collaboration requests from Creator Studio in AEMC](https://servicenow-staging.fluidtopics.net/access?context=manage-collaboration-requests&family=washingtondc&ft:locale=en-US)**

As an admin, view and approve or reject Creator Studio user requests to collaborate on an app when you use AEMC.

-   **[Added support for new Creator Studio app requests in AEMC](https://servicenow-staging.fluidtopics.net/access?context=manage-aemc-requests&family=washingtondc&ft:locale=en-US)**

As an admin, view and approve or reject new app requests from Creator Studio restricted users in AEMC.


</td></tr><tr><td>

Xanadu

</td><td>

-   **[Add users to one or more user groups](https://servicenow-staging.fluidtopics.net/access?context=manage-intake-application-requests&family=xanadu&ft:locale=en-US)**

When viewing an intake request in AEMC, add a user to one or more AES and Creator Studio user groups.

-   **[Manage user groups for Application Intake](https://servicenow-staging.fluidtopics.net/access?context=manage-app-intake-user-groups&family=xanadu&ft:locale=en-US)**

Manage which AES and Creator Studio user groups are active in your instance by updating records on the User Groups Permission Types \[sn\_app\_intake\_permission\_type\] table.

-   **[Test AEMC on a non-production instance](https://servicenow-staging.fluidtopics.net/access?context=test-aemc-non-production-instance&family=xanadu&ft:locale=en-US)**

Test AEMC on a non-production instance to ensure everything works as expected before moving to your production instance.

-   **[Open applications in ServiceNow Studio](https://servicenow-staging.fluidtopics.net/access?context=create-app-scratch&family=xanadu&ft:locale=en-US)**

As of Xanadu Store Release 2, open an application in ServiceNow Studio directly from a banner link in App Engine Studio to explore the more advanced application development environment ServiceNow Studio offers.


</td></tr><tr><td>

Yokohama

</td><td>

-   **[App Readiness and Compliance Report](https://servicenow-staging.fluidtopics.net/access?context=app-readiness-report&family=yokohama&ft:locale=en-US)**

As an App Engine Admin, use the App Readiness and Compliance report in AEMC to check if the apps you’re making are ready to go live. You can run an instance scan suite that looks for any issues that can impact app deployments. This feature helps determine if the app is prepared for use in the real world.


</td></tr><tr><td>

Zurich

No updates for this release.
