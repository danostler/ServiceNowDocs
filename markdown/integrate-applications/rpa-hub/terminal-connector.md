# Terminal \(Mainframe\) connector

Automate various actions on the IBM Personal Communications and Rocket BlueZone emulators by using the Terminal connector that RPA Desktop Design Studio provides. For example, you can simulate a command on the emulator screen or get text from the screen.

**Important:** The Terminal connector uses the IBM Personal Communications and the Rocket BlueZone APIs internally to interact and automate applications that are hosted on Mainframe or AS/400 applications. Any known issue or limitation of IBM Personal Communications and the Rocket BlueZone API is applicable.

You must first make the Terminal connector available for use on the Design surface of RPA Desktop Design Studio. To learn how to make the Terminal connector available for use, see [Use a connector in RPA Desktop Design Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/use-connector.md).

Configure the Terminal connector by capturing the IBM Personal Communications and Rocket BlueZone emulator screens and their elements on which the connector performs actions. To learn how to configure the Terminal connector, see [Configure the Terminal connector](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/configure-terminal-connector.md).

Automate actions on the IBM Personal Communications \(PCOMM\) and Rocket BlueZone emulator screens by using the methods that the Terminal connector provides. To learn how to use the methods, see [Use a component in RPA Desktop Design Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/configure-components.md).

-   **[Configure the Terminal connector](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/configure-terminal-connector.md)**  
Create automations on the IBM Personal Communications \(PCOMM\) and the Rocket BlueZone emulators by using the Terminal connector that RPA Desktop Design Studio provides. By capturing these emulator screens and their elements, you can automate actions on them.
-   **[Terminal connector methods](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/connectors-terminal-methods.md)**  
The Terminal connector methods enable you to perform various tasks on the emulator screens. For example, you can extract login data from a variable and use a method to enter the data and log in to a Mainframe application.
