# Configuring credentials in RPA Hub

In RPA Hub, you can create application credentials, robot credentials, and Time-based One-time Password \(TOTP\) authentications and link them to multiple bot processes through a credential group. Set up external credential vault to retrieve robot credentials, application credentials, or Time-based One-time Password \(TOTP\) seed.

Under credential management, you can create the following:

-   Application credentials for unattended robots to log in to the application.
-   Robot credentials for the unattended robots to log in to the system.
-   Time-based One-time Password \(TOTP\) seeds for unattended robots to seamlessly authenticate against multi-factor authentication \(MFA\)-enabled applications.
-   Credential groups for mapping application credentials and robot credentials to one or more bot processes.
-   External credential vault for retrieving robot credentials, application credentials, or Time-based One-time Password \(TOTP\) seed from an external source instead of a ServiceNow credentials record.
