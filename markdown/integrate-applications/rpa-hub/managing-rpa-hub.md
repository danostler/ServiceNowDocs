# Managing RPA Hub

Developers, support users, release managers, and administrators can use the RPA Hub application to start and stop a bot process, launch a bot process in the RPA Desktop Design Studio, execute skills, view the HashCode of a package version, and retire a bot process, package, and a robot.

-   **[Assign a bot process to a queue from Queues menu in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/assign-botprocess-queue.md)**  
Assign a bot process to a queue in RPA Hub so that it can interact with this queue. You can assign more than one bot process to a queue. Any robot that is associated with the assigned bot process can insert the work item, pick the work item, and update the work item.
-   **[Unassign a bot process from a queue in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/unassign-botprocess-queue.md)**  
Unassign a bot process from a queue in RPA Hub to stop an interaction with this queue.
-   **[Manage your work items in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/manage-work-queue-items.md)**  
Manage your work items more efficiently in RPA Hub. You can view your work items, mark them as complete, and reassign a work item.
-   **[Assign a bot process to a shared parameter in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/assign-botprocess-shared-parameter.md)**  
Assign a bot process to a shared parameter in RPA Hub so that it can interact with this shared parameter. You can assign more than one bot process to a shared parameter.
-   **[Unassign a bot process from a shared parameter in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/unassign-botprocess-shared-parameter.md)**  
Unassign a bot process from a shared parameter in RPA Hub to stop an interaction with this shared parameter.
-   **[View current robot events in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/view-robot-calendar.md)**  
View the current robot events on the **Robot Calendar** tab in RPA Hub for unattended robots. By using the calendar, you can manage and plan a robot's schedule in a single view.
-   **[Create a schedule on the robot calendar in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/create-robot-schedule.md)**  
Create a schedule on the **Robot Calendar** tab in RPA Hub to execute unattended robots. You can also identify any conflicts in the schedule by selecting the **Preview** button.
-   **[Edit a robot schedule in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/edit-robot-schedule.md)**  
Edit a robot schedule of a bot process on the **Robot Calendar** tab in RPA Hub to resolve any scheduling conflicts or to modify the details on the schedule form.
-   **[Delete a robot schedule in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/delete-robot-schedule.md)**  
Delete an existing robot schedule of a bot process on the **Robot Calendar** tab in RPA Hub that you no longer need.
-   **[Manage scheduled maintenance days in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/manage-sm-request-rpa.md)**  
Track scheduled maintenance days on the **Robot Calendar** tab to manage your current and future automations.
-   **[View the code quality rules list in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/view-cqr-rpa.md)**  
View a list of the pre-defined code quality check rules along with their active status and severity.
-   **[Update a code quality rules form in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/edit-cqr-rpa.md)**  
Update few fields in the code quality rule form to set the code quality check standard.
-   **[Download a package version in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/download-package-version.md)**  
Download a package version in RPA Hub so that it can be opened from the RPA Desktop Design Studio to build, debug, or modify the automation.
-   **[Upload an automation package attachment in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/upload-package-version-attach.md)**  
Upload a zip file of the automation package in RPA Hub to manually migrate the package from a lower \(non-production\) to a higher \(production\) environment.
-   **[Import a package version attachment in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/import-attachment-package-version.md)**  
Import package version attachments automatically instead of manually uploading a package version attachment, by selecting the **Import Attachment** button on a package version.
-   **[Override a skill package version for multiple bot processes in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/override-skill-version-multi-bot-process.md)**  
Override a skill package version in RPA Hub for multiple bot processes that are associated with the same attended or unattended package. You can select the latest version of the skill package without publishing the attended or unattended package again from RPA Desktop Design Studio.
-   **[Verify the HashCode of a package version in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/view-hash-code.md)**  
Verify the HashCode of a package version in RPA Hub so that you can validate the HashCode of the manually uploaded automation zip file with the associated package version record.
-   **[Edit an unattended robot in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/edit-robot.md)**  
Edit an existing unattended robot in RPA Hub to modify a few details of the robot, convert the robot to a high density robot,and view the robot state.
-   **[Edit a package in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/edit-package.md)**  
Edit an existing package in RPA Hub to modify the details such as the name or description.
-   **[Edit a bot process in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/edit-botprocess.md)**  
Edit an existing bot process in RPA Hub to modify a few details.
-   **[Start an unattended bot process in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/start-bot-process.md)**  
Start an unattended bot process in the specified robot machine to execute the automation by the selected robot.
-   **[Stop an unattended bot process in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/stop-bot-process.md)**  
Stop the unattended bot process to abort the automation by the selected robot.If robot pool is enabled for a particular bot process, then the execution is stopped for all the robots of the robot pool.
-   **[Launch in Studio from RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/launch-studio.md)**  
Review, edit, and debug automations by opening the associated package version of the bot process in the RPA Desktop Design Studio.
-   **[Unassign an attended user or group from an attended bot process in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/unassign-rda-users.md)**  
Unassign an attended user or a group from an attended bot process if the user is moving out of the business process.
-   **[Invoke Embedded Task Automation via API in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/create-button-att-config-rpa.md)**  
Invoke an attended bot process \(attended automation\) from the UI experiences by creating an action.
-   **[Activate an attended configuration record in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/activate-attend-config-rpa.md)**  
Activate an attended configuration record to enable the UI action that triggers the attended automation on the associated ServiceNow form.
-   **[Deactivate an attended configuration record in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/deactivate-attend-config-rpa.md)**  
Deactivate an attended configuration to move the record to inactive state.
-   **[Edit an attended configuration record in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/edit-attended-config-rpa.md)**  
Edit an existing attended configuration record in RPA Hub to modify a few details of the record.
-   **[Delete an attended configuration record in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/delete-attended-config-rpa.md)**  
Delete an attended configuration record that you no longer need.
-   **[View a package version in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/view-package-version.md)**  
View a package version in RPA Hub so that it can be opened from the RPA Desktop Design Studio to build, debug, or modify the automation.
-   **[View alerts in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/view-alerts-rpa.md)**  
View notifications that are generated by the alert rules. For example, you can view alerts that pertain to robots, schedules, process jobs, and so on.
-   **[View installed RPA Hub plugins](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/view-installed-rpa-plugins.md)**  
View a list of available RPA Hub plugins, the plugin versions, and dependency packs to verify the availability of a specific version that you want to install and use on the RPA Desktop Design Studio.
-   **[View process jobs for all bot processes in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/view-process-jobs.md)**  
View a list of execution logs for all bot processes.
-   **[View maintenance requests in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/view-mr-rpa.md)**  
After a scheduled maintenance event card is accepted or rejected on the robot calendar, a maintenance request is created for tracking.
-   **[View automation logs in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/view-automation-logs-rpa.md)**  
Automation logs is series of succession stages or a series of checkpoints that occur during the execution of a bot process. It enables monitoring and auditing the automation process. These logs are generated from the robot for unattended and attended bot processes.
-   **[Delete a TOTP authenticator in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/delete-mfa-authenticator-rpa.md)**  
Delete a Time-based One-time Password \(TOTP\) authenticator record in RPA Hub so that you can dissociate it from a bot process.
-   **[Retire an MFA authenticator in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/retire-mfa-authenticator.md)**  
Retire a multi-factor authentication \(MFA\) authenticator in RPA Hub so that you can turn off the authentication. After you retire an MFA authenticator, it can’t be used again.
-   **[Retire a bot process in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/retire-bot-process.md)**  
Retire a bot process to turn it off. It's one of the life-cycle stages of a bot process. After you retire a bot process, you can't use it anymore.
-   **[Retire a package in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/retire-package.md)**  
Retire a package to turn it off. It is one of the life-cycle stages of a package. After you retire a package, it cannot be used further.
-   **[Retire a package version in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/retire-package-version.md)**  
Retire a package version in RPA Hub so that the version can't be used again. After a package version is retired, it can’t be used again.
-   **[Retire a robot in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/retire-robot.md)**  
Retire a robot to turn it off. It is one of the life-cycle stages of a robot. After you retire a robot, it cannot be used further.
-   **[Enable the client callable option on a flow or subflow in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/enable-client-flow-rpa.md)**  
Enable a client script to trigger an RPA Hub flow or subflow.
-   **[Configure the system properties for an activity stream in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/configure-sys-properties-activity.md)**  
Configure the system properties for an activity stream of work queues, schedules, packages, process parameters, and shared parameters in RPA Hub. You can add or remove the fields that are captured in the work notes of an activity stream.
-   **[Display an activity stream for bot processes and robots in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/display-activity-stream-sys-property-rpahub.md)**  
Configure the system property to display an activity stream for bot processes and robots in RPA Hub.
-   **[Verify the associated schedules, parameters, packages, and queues in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/verify-botprocess-tasks.md)**  
Verify that the bot process has the associated schedule, process parameters, shared parameters, package, package version, and work queues in RPA Hub. You do this task when you’re migrating your data from a lower \(non-production\) environment to a higher \(production\) environment.
-   **[Mark the bot process as ready for migration in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/mark-bot-process-ready-for-migration.md)**  
Mark the bot process as ready for migration in RPA Hub so that the bot process configuration and associated assets can be migrated to another environment. With this feature, you can create an automation and test it in a lower environment and then migrate it to a higher or production environment.
-   **[Migrate the bot process configuration to another environment in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/migrate-bot-process.md)**  
Migrate the bot process configuration and associated assets to another environment in RPA Hub. With this feature, you can create an automation and test it in a lower environment and then migrate it to a higher or production environment.
-   **[Convert existing unattended robots to high density robots in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/integrate-applications/rpa-hub/convert-robot-hdr.md)**  
Run multiple unattended automations simultaneously on a single machine by batch converting one or more existing robots to high density robots using the **Actions on selected rows** option.
