# View diagram

After creating a diagram, you can view it to edit or continue creating an automation from it. A diagram is a visual representation of the automation that you want to create.

## Before you begin

Role required: sn\_ac.automation\_technical\_user, playbook.write, playbook.activity\_def\_read, connection\_admin, or cmdb\_read

**Note:** You should have a diagram ready to so that you can view it.

## Procedure

1.  Navigate to **Workspaces** &gt; **Automation Center Workspace**.

2.  On the **Lists** tab \(\[Omitted image "autocenter-list-icon.png"\] Alt text: List icon\), under **Build**, select **All Automation Requests**.

3.  Select an automation request for which you want to view the diagram.

4.  Select **View diagram**.

5.  The Requirement Intake page is displayed.

    The page displays two views of the requirement diagram:

    -   Diagram view

        \[Omitted image "diagram-view.png"\] Alt text: Diagram view

    -   Board view

        \[Omitted image "board-view.png"\] Alt text: Board view

6.  You can edit the diagram if required.

    For more information about editing or creating diagrams, see .

    When viewing or editing a diagram, we need not set a trigger. A trigger will be set when the automation is created from the requirement diagram.
