# Shared parameters in RPA Hub

Shared parameters enable you to store the global variables or configurable items in one place. These variables are used across bot processes.

Instead of hard-coding these values directly into each individual bot process, the shared parameters are stored in a central location, making it easier to update them universally and avoid duplication.

You cannot delete a shared parameter that is associated to a published bot process.

Shared parameters offer numerous benefits such as consistency, centralized management that contributes to efficient and maintainable development practices.

-   **[Create a shared parameter in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/create-shared-parameter.md)**  
Create a shared parameter so that you can store the global variables or configurable items in one place. These variables are used across bot processes.
-   **[Assign a bot process to a shared parameter in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/assign-botprocess-shared-parameter.md)**  
Assign a bot process to a shared parameter in RPA Hub so that it can interact with this shared parameter. You can assign more than one bot process to a shared parameter.
-   **[Unassign a bot process from a shared parameter in RPA Hub](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/unassign-botprocess-shared-parameter.md)**  
Unassign a bot process from a shared parameter in RPA Hub to stop an interaction with this shared parameter.
