# Request access to data assets

Request access to data products and data interfaces in the Workflow Data Fabric to consume data for your workflows and applications.

## Before you begin

Role required: WDF Consumer \(wdf\_consumer\)

## About this task

Use the data catalog to discover and request access to data products and data interfaces that contain the data you need for your workflows. Data stewards review and approve access requests based on data governance policies and business requirements.

**Note:** This feature is only available for data interfaces and data products and not other data assets.

## Procedure

1.  Navigate to **All** &gt; **Workflow Data Fabric** &gt; **Workflow Data Fabric Home**.

2.  Select the Data catalog icon in the left sidebar.

    The data catalog displays all available data assets that you can browse and search.

3.  Browse or search for the data product or data interface you want to access.

    You can filter by data domain, category, or use the search functionality to find specific data assets.

4.  Select the data product or data interface you want to request access to.

    The data asset details page opens, displaying metadata, schema information, data lineage, and access controls.

5.  Select **Request access**.

    \[Omitted image "dc-data-asset-request-access.gif"\] Alt text: Data asset page showing Request Access button.Your access request is submitted to the data steward for review and approval.


## Result

The data steward receives a notification about your access request and can review, approve, or deny the request. You receive an email notification when the request status changes.
