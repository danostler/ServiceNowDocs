# Microsoft Outlook

The Microsoft Outlook connector enables you to automate standard actions on the Microsoft Outlook application as part of an automation. For example, you can automate sending or replying to emails.

The Microsoft Outlook connector exposes methods to automate the actions on Microsoft Outlook. These methods can work with other methods or components in an automation.

To use the Microsoft Outlook connector, see [Use a connector in RPA Desktop Design Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/use-connector.md).

To expose the Microsoft Outlook methods, see [Use connector method](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/use-connector-method.md).

To use the Microsoft Outlook connector methods, see [Use a component in RPA Desktop Design Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/configure-components.md).

**Note:** 32 bit Microsoft Office installations are not supported.

-   **[Microsoft Outlook connector methods](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/microsoft-outlook-connector-methods.md)**  
The Microsoft Outlook connector methods automate various actions on the Microsoft Outlook. In an automation flow, the methods are connected with other methods and components. To use the methods, you must first expose them.
