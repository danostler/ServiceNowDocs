# Building automations

Create automation projects, use components, and create the activities and variables that are required to design automations in RPA Desktop Design Studio.

-   **[Create an automation project manually](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/create-automation-project.md)**  
Create an attended or unattended automation project manually in RPA Desktop Design Studio that enables you to automate tasks. You can create an automation project that is based on a prebuilt template and targeted at a use case, or you can create an automation project without using a template.
-   **[Create an automation project from a template](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/create-automation-with-template.md)**  
Create an automation project from the default automation templates that RPA Desktop Design Studio provides or the templates that you have designed on the RPA Desktop Design Studio and published to the RPA Hub. An automation template caters to a use case and provides a framework of automation.
-   **[Create a skill project in RPA Desktop Design Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/create-skill-project.md)**  
Create a skill automation project in RPA Desktop Design Studio that can be reused in other automations.
-   **[Open a local automation project in RPA Desktop Design Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/open-project-rpa-studio.md)**  
Open an automation project from the local file system in RPA Desktop Design Studio, so that you can debug, build, or modify the automation.
-   **[Open a cloud automation project in RPA Desktop Design Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/open-project-remote-rpa-studio.md)**  
Open an automation project remotely from RPA Hub in RPA Desktop Design Studio, so that you can debug, build, or modify the automation.
-   **[Manage plugins in RPA Desktop Design Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/install-plugins-rpa-studio.md)**  
Install, update, or remove plugins to design automations while you're creating unattended and attended automation projects in RPA Desktop Design Studio.
-   **[Create user plugins for RPA Desktop Design Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/install-user-plugins-rpa-studio.md)**  
Create user plugins by using Microsoft Visual Studio to enable RPA developers to use complex .NET components or libraries in RPA Desktop Design Studio in a seamless way.
-   **[RPA Desktop Design Studio ports](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/overview-ports-rpa-studio.md)**  
Components in the RPA Desktop Design Studio have ports that are used while creating an automation workflow.
-   **[Create and use an activity in RPA Desktop Design Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/configure-activity-rpa-studio.md)**  
Create and use an activity to provide modularity to an automation project in RPA Desktop Design Studio.
-   **[Create and use a variable in RPA Desktop Design Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/configure-variables-rpa-studio.md)**  
Create a variable in RPA Desktop Design Studio.
-   **[Create an attended automation form in RPA Desktop Design Studio](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/create-rda-form.md)**  
Design attended automations by creating an attended user interface \(UI\) form in RPA Desktop Design Studio.
-   **[Open Service and Runtime logs](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/open-logs-rpa-studio.md)**  
Logs record events that occur during the execution of an automation process. These logs are crucial for debugging and monitoring the performance of the robots.
-   **[Post requisites after installing a hot fix](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/integrate-applications/workflow-data-fabric/post-req-hot-fix-rpa.md)**  
Perform these tasks after installing a hot fix for either Universal App Connector \(UAC\) or Chromium connector on your instance to verify the hot fix is accurately downloaded.
