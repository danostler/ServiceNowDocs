# ServiceNow MCP Registry

Model Context Protocol \(MCP\) enables AI agents to connect to external systems and perform the required actions without any human intervention. ServiceNow MCP Registry is a private registry where large language models \(LLMs\), AI agents, and teams discover and connect to approved MCP servers, finding the best tool for every business problem, governed at every step. With these configurations, you can use any generative AI assistant to perform the required actions in ServiceNow.

## AI-native discovery

The Registry enables AI models and humans query available MCP servers and select the ones best suited to solve a business problem. Every server exposed to an LLM is pre-approved in AICT, helping to ensure autonomous decisions never bypass enterprise governance. It works with AI Agent Studio, Claude, and so on.

## Governed server catalog

The Registry unifies three server types into a single trusted catalog:

-   MCP servers built by business units or the WDF abstraction layer.
-   Customer servers published through the MCP Server Console or discovered through the CMDB.
-   Third-party certified servers for seamless business unit agentic workflow execution.

## Agentic workflow support

Business unit agentic workflows need their MCP servers trusted and available before execution begins. Pre-requesting approvals for all servers through AICT and centralizing them in the registry eliminates per-run approval overhead and helps ensure every workflow runs without connectivity surprises.
