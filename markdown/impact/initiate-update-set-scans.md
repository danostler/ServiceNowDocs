# Initiate update set scans

You can scan open update sets for findings to get insights into what you are importing and exporting across your environments. 

## Before you begin

Role required:

-   Scan Engine users can initiate scans.
-   System Admins have read/write access to the custom application table.

**Note:** Local update set scans are performed in batches, which means all update sets \(parent or child\) in the grouping will be scanned. Retrieved update set scans are scanned individually, not in batches. 

## Procedure

1.  Navigate to **All** &gt; **System Update Sets**, and then select one of the following:

    -   **Local Update Sets**
    -   **Retrieved Update Sets**
2.  Select the name of the update set you want to scan.

3.  In **Related Links**, select **Update Set Scan**.
