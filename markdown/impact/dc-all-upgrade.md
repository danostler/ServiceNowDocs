# Upgrading the Data Collection Content Pack

When you upgrade the Data Collection Content Pack, you must account for any customizations you've made.

Since users might change artifacts within this app \(for example, indicator, indicator source, or data collection job\), remember that the upgrade process would be the same as the process for upgrading the platform or any other store app. The upgrade process skips customized \(or changed\) application files. You can view unchanged customizations in the upgrade history, and do one of the following:

-   Accept the customizations.
-   Revert the customizations back to the base system, and then reapply them.
