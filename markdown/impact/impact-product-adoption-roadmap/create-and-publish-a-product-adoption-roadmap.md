# Manage product adoption roadmaps for Impact Delivery Instance

Create a product Adoption Roadmap \(PAR\) with a single click by selecting the template that best supports your specific business outcome.

## Before you begin

Role required: Impact admin, Platform Owner, Impact Portfolio Owner, and Impact Executive.

## Procedure

1.  Navigate to **Impact** &gt; **Product adoption** &gt; **Product adoption roadmaps**.

2.  Select **Create a product adoption roadmap**.

3.  Select an instance if there are multiple instances available.

4.  Enter the **Owner**.

5.  In **How do you want to start?**, complete any of the following actions.

    1.  Select **Notes** to view static and phase specific notes for a capability.

        You can view static and phase-specific note sections under **Notes**. Static notes display for the selected capability regardless of the phase in which they are added.

        \[Omitted image "capability-notes-pars.png"\] Alt text: Select Notes to view the static and phase specific notes for a capability.

    2.  Select **Notes specific to this phase** to display additional information tailored for the capability and phase including prerequisites, features, training, and planned activities.

        You can find resources related to adopting the capability under **Related Resources**.

        \[Omitted image "Pars-notes-specific-to-phase.png"\] Alt text: View notes specific to this phase.

    3.  Select **Start an initiative** to create an initiative specific to this capability.

7.  Select **Edit Roadmap Details** to edit the name, description, and owner of the map.

8.  Select **Manage capabilities** for a phase to add or remove capabilities.

9.  Select the ellipsis icon for a phase to modify a phase and do the following.

    1.  To delete a phase, select **Delete phase**.

    2.  To edit phase titles and descriptions, select **Edit titles/descriptions**.

    \[Omitted image "Pars-roadmap-Phase-options.png"\] Alt text: Modify a phase using the ellipsis icon next to a capability.

10. Select **Publish**.

    The product adoption roadmap is published and is available to other members on your account and the impact squad.


## What to do next

You can also edit the published roadmap by selecting **Edit roadmap** to manage the adoption phases and capabilities.
