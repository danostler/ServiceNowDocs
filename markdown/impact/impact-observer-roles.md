# Roles installed with Instance Observer

Several types of components are installed with activation of the Impact plugin, including tables, user roles, and scheduled jobs that are used in Instance Observer.

## Roles installed

<table id="table_kk4_krw_chc"><thead><tr><th>

Role title \[name\]

Description

IO Standard\[io\_standard\]

</td><td>

-   Grants full access to all the features such as performance dashboards, reports, alert triage, and availability insights within Instance Observer.
-   Allows users to fully utilize the platform’s capabilities.

</td><td>

None

</td></tr><tr><td>

IO read only\[io\_readonly\]

</td><td>

Grants limited read-only access to the Instance Observer.

 Allows users to view features such as configured and unconfigured alerts, rules, and integrations. However, users are restricted in performing actions that involve configuration changes. The restrictions include:

-   Creating or editing alert rules
-   Managing integration
-   Scheduling reports

</td><td>

None
