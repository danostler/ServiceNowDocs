# Estimate, review, and collect fees for an information request case using Information Request Playbook

In this step, you will estimate and collect fees.

## Estimate fees needed for an information request case

Use the fee estimator to estimate the fees associated with this request.

### Before you begin

Role required: admin

### Procedure

1.  Enter the following inputs into the fee estimator:

    -   How much time is needed for the request
    -   How many people are needed for this request
    -   How many pages are needed for this request
    **Note:** The fee estimator can only be run once.

2.  Select **Next**.

3.  Review the calculated output.

4.  Use the calculated output as the estimated fee for the case, and enter details on the fee estimate in the work notes.

    1.  Select **Add to Case**, noting that the Estimated fees field has been updated.
    2.  Enter the Fees calculated in Work notes.
    If the requester qualifies for a fee waiver, enter that information here.

5.  Select **Request Fee Approval**.

    The fee estimate is now sent to billing to be approved. A billing group must already created for it to be routed to them in this step. For information on .

    For more information on how to approve a fee estimate as an approve, see . Once the fee estimate has been approved by the approver, mark the step as complete.

6.  Select **Send for Requester Approval** to send the fee estimate to the requester.

    This activity will automatically be marked complete once the requester has approved the fee, and the case will be moved to the **Process Payment** activity.


## Document a completed payment for an information request case in Information Request Playbook

Verify and document that payment has been sent by the requester in the Process payment activity of the playbook.

### About this task

Estimate fees in the last step, send to requester. This is what they will pay.

### Before you begin

Role required: admin

### Procedure

1.  Verify the payment has been sent, and select **Move to Process**.


### Result

The Review stage is now complete and the case is moved to the Process stage, where the requested documents can be processed and reviewed by several teams before they can be routed to the requester.
