# Configure the security classification of an evidence record in Investigative Case Management

You can change and configure the security classification of an evidence record for labeling purposes.

## Before you begin

Role required: icm.investigator, supervisory\_agent, admin

## About this task

**Note:** A user without write access to evidence cannot create a Chain of Custody log.

## Procedure

1.  Navigate to the CSM Configurable Workspace and select **My active cases**.

    Alternatively, you can select a case using the CSM Configurable Workspace Lists menu by selecting **Lists** &gt; **Investigative Cases** &gt; **All** and selecting the case.

2.  Select the case you wish to update the evidence security classification for.

    You can only edit the security classification for one evidence record at a time.

3.  Select the **Evidence** tab, and select the evidence record you wish to update the evidence security classification for.

4.  In the **Security Classification** dropdown, select one of the following:

    -   Public
    -   Internal Use Only
    -   Confidential
    -   Restricted
    -   Law enforcement sensitive \(LES\)
    -   Privileged
    -   Personally Identifiable Information \(PII\)
    -   Protected Health Information \(PHI\)
    -   Classified Confidential
    -   Classified Secret
    -   Classified Top Secret
    -   For Official Use Only \(FOUO\)
    -   Evidence Under Seal
5.  Select whether or not you need to restrict access to the evidence record.

6.  Select **Save**.
