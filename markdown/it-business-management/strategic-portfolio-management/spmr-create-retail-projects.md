# Create a retail project

Set up a retail project from a template to plan and track store operations, including store openings, relocations, refurbishments, technology refreshes, and closures.

## Before you begin

Role required: sn\_spm\_retail.project\_manager

## Procedure

1.  Navigate to **Workspaces** &gt; **Project Workspace**.

2.  Create a retail project using one of the options.

    -   Select **New project**.
    -   Select **New project from template**.
3.  Select **Customer Project** from the Project class.

4.  Select a project template using the **Project type** list that matches the type of store work from the following options.

    -   **Retail New Store Opening** to open a new store.
    -   **Retail Store Relocation** to move an existing store to a new location.
    -   **Retail Store Refurbishment** to refurbish an existing store.
    -   **Retail Store Technology Refresh** to upgrade in-store technology, for example POS systems, security cameras, or network connectivity.
    -   **Retail Store Closure** to close a store.
5.  Enter a unique name in the **Project name** field.

6.  Select the planned date to start work using the **Planned start date** calendar palette.

7.  Select **Confirm**.

8.  Select the **Details** menu from the menu.

9.  Select the name of the store for which the project is being executed using the **Service organization** field.


## Result

A project is created based on the selected project template with tailored project task and playbook. You will see a the Planning page with prepopulated project tasks and their dependent tasks.

## What to do next

Enter the key dates of your work in Project Type list from the Details tab. For more details, see [Enter project type fields](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/it-business-management/strategic-portfolio-management/spmr-project-type-fields.md).
