# Data migrated from demands to Enterprise Agile Planning \(EAP\) entities

Data fields are migrated from a demand to an EAP entity when the entity is created from the demand.

<table id="table_ujx_fxg_vfc"><thead><tr><th>

Field type

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Demand

</td><td>

The following demand fields are migrated to the created entity:-   Business Capabilities
-   Product
-   Primary target
-   Team
-   Demand manager
-   Description
-   Department
-   Priority
-   Approved start date
-   Approved end date
-   Short description
-   Primary goal
-   Strategic program
-   Service offering
-   Business service
-   Business applications
-   Score
-   Requested by
-   Capital expense
-   Operating expense
-   Start date
-   Program
-   Total planned cost
-   Investment Class
-   Portfolio
-   Investment Type
-   ROI %
-   Rate Model
-   Model ID
-   Expense type
-   Financial benefit

</td></tr><tr><td>

Resource assignments

</td><td>

The resource assignments of a demand are migrated to the created entity.

</td></tr><tr><td>

Financial records

</td><td>

Cost plans and breakdowns, benefit plans and breakdowns \(monetary and non-monetary\), and investment budgets are migrated to the created entity.

</td></tr><tr><td>

Attachments

</td><td>

Demand attachments.

</td></tr><tr><td>

Goal/target relationships

</td><td>

Goal/target relationships of the demand.

</td></tr></tbody>
