# Highlighted Value Condition form

Learn about the fields of the Highlighted Value Conditions form. Use this form to configure the display of highlighted fields in Strategic Planning workspace.

<table id="table_dyb_41y_2wb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Condition

</td><td>

Condition for the field value.For example, Status is Green.

</td></tr><tr><td>

Color

</td><td>

Color for the field value that meets the condition specified.

</td></tr><tr><td>

Application

</td><td>

Scope of yourServiceNow instance. By default, this field is set to **Portfolio Planning**.

</td></tr><tr><td>

Order

</td><td>

Order in which this color appears on the legend in the Hierarchy view of the Prioritization page.

 For example, for the Planning Status column, the value **New** is given the order 1.

 \[Omitted image "hierarchy-legend.png"\] Alt text: Hierarchy view legend

</td></tr><tr><td>

Hide Label

</td><td>

Option to hide the label for the highlighted value condition. When selected, the Label field is not available.

</td></tr><tr><td>

Label

</td><td>

Name of the label displayed for the highlighted value condition. This field is not available when **Hide Label** is selected.

</td></tr><tr><td>

Show Icon

</td><td>

Option to show icon beside the field value.

</td></tr><tr><td>

Icon

</td><td>

Icon that is displayed beside the field value. This field is shown only if the **Show Icon** field is selected.

</td></tr><tr><td>

Variant

</td><td>

Option to define color variant. The available options are **primary**, **secondary**, and **tertiary**.

</td></tr><tr><td>

Value Override

</td><td>

Custom display text that replaces the actual field value in the view. When specified, the override text appears instead of the stored value.

</td></tr></tbody>
