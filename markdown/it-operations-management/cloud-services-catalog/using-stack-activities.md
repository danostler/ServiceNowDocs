# Stack activities view

Display the cloud actions on your stacks and resources by using the **View Activities** tab.

You can do multiple actions in the stacks life cycle such as track operations, view activities, view dependency, and do actions. The following example shows the stack activities that you can perform and the following table describes the activities.

\[Omitted image "using-stack-activities.png"\] Alt text: Stack activities view.

<table id="table_wfx_dsg_h1c"><thead><tr><th>

Activity

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Change Requests

</td><td>

Change requests that have been submitted for stacks or resources that are assigned to you. The system generates change requests when you perform any life-cycle operations on a stack or resource.

</td></tr><tr><td>

Incidents

</td><td>

Incidents for stacks and their associated resources that are assigned to you.

</td></tr><tr><td>

Tasks

</td><td>

Catalog tasks that are created when a request for a stack fails to provision.

</td></tr><tr><td>

Scheduled Operations

</td><td>

List of scheduled operations that are associated with stacks and are displayed as these features:-   Operation Time
-   Target
-   Operation
-   Status
