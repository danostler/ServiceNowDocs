# Alert field mapping on the Respond page

The fields to be configured on the Respond page, when selecting an action to be triggered by an alert automation.

<table id="table_kpr_zpx_t1c"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Incident field name

</td><td>

Name of the field that will appear in the incident.

</td></tr><tr><td>

Map field

</td><td>

Option that indicates how the incident field is to be mapped:-   **Same as**: The selected incident field will have the same value as the indicated alert field.
-   **To**: The selected incident field will be set to a specified value.

</td></tr><tr><td>

Alert field value

</td><td>

Value that will be set in the alert field:-   If you select **Same as** in the Map field column, the alert field value that you want to appear in the incident field also appears in the Incident field name column.
-   If you select **To** in the Map field column, enter the alert field value that you want to appear in the specified incident field.

</td></tr></tbody>
</table>To return to the procedure, see [Create Respond automation](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/it-operations-management/service-operations-workspace-for-itom-apps/respond-alert-sow-itom.md).
