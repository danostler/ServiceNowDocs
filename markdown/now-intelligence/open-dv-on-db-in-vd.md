# Open a visualization in the Visualization Designer from a dashboard

From the dashboard editor, open a data visualization in the Visualization Designer. You have a different set of options.

## Before you begin

The data visualization must exist in the data visualization library.

Role required: itil, report\_user, viz\_creator, or higher, if you own the data visualization. viz\_admin or higher for any data visualization. You also need the right to edit the dashboard.

## About this task

While working in the dashboard editor, you may find that you want to reconfigure a data visualization in a way that is only possible in the Visualization Designer, not in the dashboard editor. For example, you might want to configure the **Drill down to chart** interaction. The **Actions** menu on the data visualization lets you open the visualization in the Visualization Designer in one step, without lots of navigating.

## Procedure

1.  Navigate to **Platform Analytics** &gt; **Dashboards** and open your dashboard.

2.  Select **Edit** to put the dashboard in edit mode.

3.  Add the data visualization if it is not already there.

    For more information, see [Edit in-line Platform Analytics dashboard elements](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/now-intelligence/edit-db-elements-in-ac.md).

4.  On the data visualization, expand the **Actions** menu.

    \[Omitted image "dv-on-db-edit-vd.png"\] Alt text: Actions menu on data visualization, showing Edit in designer action.

5.  Select **Edit in designer**.


## What to do next

Proceed to edit the data visualization as described in [Edit a data visualization in the Visualization Designer](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/now-intelligence/edit-dv-ac-center.md).
