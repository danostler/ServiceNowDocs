# Duplicate an exploration

Create multiple copies of a AI Data Explorer exploration to repeat analyses, give yourself editing rights without affecting the original, or experiment with multiple variations of an exploration.

## Before you begin

Role required: now\_assist\_explorer\_user, and you must have the exploration shared with you at least with viewing rights.

## About this task

Duplicating an exploration grants the following possibilities:

-   Create repeating analyses on the basis of an existing exploration. For example, you see an exploration about trends in May. You want to repeat the same analysis for June. By duplicating the May exploration, you don't need to start the exploration flow from scratch.
-   Duplicate an exploration where you have only read-only access, to create a copy that you can edit. With editing rights, you can refine existing questions or ask new ones.
-   Create a polished version of an exploration. To gather the right insights to drive decision making, you may need a sandbox where you can try out different takes and sets of questions. You can duplicate an exploration and experiment with the duplicates without losing the original detailed analysis.

## Procedure

1.  Launch AI Data Explorer.

    See [Launch AI Data Explorer](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/now-intelligence/platform-analytics/launch-now-assist-explorer.md).

2.  Locate and open the exploration that you want to duplicate.

    See [Open an existing exploration](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/now-intelligence/platform-analytics/open-existing-exploration.md).

3.  Expand the additional actions list and select **Duplicate**.

    \[Omitted image "nowass-expl-duplicate-action.png"\] Alt text: The Duplicate action.

4.  In the Duplicate exploration dialog, set a name for the duplicate exploration and decide whether you want it to have the same access permissions as the original.

    If you select **Copy with access permissions**, the users who had the original exploration shared with them have the same access to the duplicate. You have editing access to the duplicate whether or not you could edit the original, regardless of the choice you make here.

5.  Press **Duplicate**.
