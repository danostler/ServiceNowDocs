# Copy a project

Copy an existing definition to apply its related configurations to a new project definition.

## Before you begin

Role required: sn\_process\_mining\_analyst, sn\_process\_mining\_power\_user, or sn\_process\_mining\_admin

## Procedure

1.  Navigate to **All** &gt; **Process Mining** &gt; **Process Mining Workspace**.

2.  From the Projects list, select the ellipses beside the project you want to copy.

3.  Select the **Copy**.

4.  Select **OK** to confirm you want to copy the configuration and create a project definition from it.

    A new process model definition creates with the following:

    -   same description
    -   source type
    -   filter conditions
    -   activity definitions
    -   breakdown definitions
    -   child table definitions
    -   improvement opportunities
5.  Change the name, description, filter conditions, or definitions as needed, and select **Update**.
