# Rule-based finding definition form from Classic view

Use the Finding Definition form to configure rule-based finding definitions to deliver discovered insights to your Summary and insights page.

<table id="id_km4_j3g_kvb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Class

</td><td>

Whether this finding definition applies to a **Process** or a **Project**.

</td></tr><tr><td>

Active

</td><td>

Whether the finding definition is active.

</td></tr><tr><td>

Process Configuration

</td><td>

The process configuration associated with this finding definition. This field is automatically populated.**Note:** This field appears only when you are creating an improvement opportunity for a process table.

</td></tr><tr><td>

Project

</td><td>

The project associated with this finding definition.

 **Note:** This field appears only when you are creating a finding definition for a project.

</td></tr><tr><td>

Type

</td><td>

Type of finding. You can select from the list available.

</td></tr><tr><td>

Category

</td><td>

Category for this finding definition.-   Quality
-   Automation
-   Conformance
-   Performance

</td></tr><tr><td>

Message

</td><td>

Title for the insight. This text appears at the top of the insight card on the Summary and insights page.

</td></tr><tr><td>

Select impacted KPIs

</td><td>

Select the KPIs most likely impacted by the findings. Choosing the affected KPIs enables you to assess the business impact. You can view the KPI-related findings in the Insights panel of the Process Mining workspace.

</td></tr></tbody>
