# Use cases

View a list of example use cases for CPQ.

-   **[Use case: Embed CPQ UI in an HTML page](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/order-management/sales-and-order-management/use_case_embed_logik_io_ui_in_an_html_page.md)**  
Learn how to embed the CPQ user interface in an HTML iFrame.
-   **[Use case: Embed CPQ UI in a Salesforce VisualForce page](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/order-management/sales-and-order-management/use_case_embed_logik_io_ui_in_salesforce_visualforce_page.md)**  
Learn how to embed the CPQ user interface in a Salesforce VisualForce page.
-   **[Use case: Embed CPQ UI in a Lightning Web Component](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/order-management/sales-and-order-management/use_case_embed_logik_in_a_lightning_web_component.md)**  
Learn how to embed CPQ in a LWC in a Salesforce org.
-   **[Use case: Populate display names and quote lines in Configuration Field Data Set objects](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/order-management/sales-and-order-management/use_case_populate_display_name_and_quote_line_in_configuration_field_data_sets.md)**  
Learn how to set up custom objects and Apex triggers in SFDC to populate the display names of CPQ fields.
-   **[Use case: Pairing set repeaters and visualization components](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/order-management/sales-and-order-management/use-case-pairing-set-repeaters-and-visualization-components.md)**  
Learn how to configure an experience where the user must answer the same questions several times so that the user is presented with a rendering of the currently configured solution.
-   **[Use case: Configuration line item to quote line flow](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/order-management/sales-and-order-management/use-case-configuration-line-item-to-quote-line-flow.md)**  
By making a few adjustments to a flow template includes with the CPQ Extension package version 1.8 or later, you can parse the extended information from a configuration and map it to custom fields without using a QCP script.
-   **[Use case: Using the Salesforce Quote Calculator plugin to integrate data from CPQ to Salesforce quotes and quote lines](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/order-management/sales-and-order-management/integrate_config_data_from_productlist_extended_to_salesforce_quote_and_quote_lines_using_quote_calculator_pluginqcp.md)**  
In the CPQ Extension for Salesforce CPQ package version 1.7 or earlier, use the Salesforce Quote Calculator Plugin to parse extended information from a configuration and map it to custom fields.
-   **[Use case: Using Apex to call or test CPQ APIs](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/order-management/sales-and-order-management/use_case_how_to_use_apex_to_calltest_logik_apis.md)**  
Learn how to use Apex to call or to test CPQ APIs.
-   **[Use case: Displaying the CPQ native UI via direct URL](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/order-management/sales-and-order-management/use_case_display_logik_io_native_ui_via_direct_url.md)**  
Learn how to initialize a configuration UI using a URL instead of an external library.
-   **[Use case: Modifying layout text with SLDS styles](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/order-management/sales-and-order-management/use_case_modifying_layout_text_with_slds_styles.md)**  
Learn how to implement styles in the Salesforce Lightning Design System to control the appearance of text.
