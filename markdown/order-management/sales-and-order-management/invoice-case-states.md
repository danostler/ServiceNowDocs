# Invoice case states on Business Portal

An invoice case moves through different states during its life cycle after an accounts payable specialist processes it.

<table id="table_owz_fk3_lcc"><thead><tr><th>

State

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Draft

</td><td>

The invoice case has been created.

</td></tr><tr><td>

New

</td><td>

The invoice case has been submitted for processing.

</td></tr><tr><td>

Work in Progress

</td><td>

The assigned agent is working on the invoice case.

</td></tr><tr><td>

Awaiting Info

</td><td>

The assigned agent is waiting for the named contact to provide requested information for the invoice case.

</td></tr><tr><td>

Resolved

</td><td>

The assigned agent has proposed a resolution for the invoice case to the contact.

</td></tr><tr><td>

Closed

</td><td>

The invoice case has been resolved and the resolution has been accepted by the contact.When all the case lines for an invoice case are in a Final state, the case can be set to Closed. Final states include:

-   Resolved - Accepted
-   Resolved - Denied
-   Canceled

</td></tr><tr><td>

Cancelled

</td><td>

The invoice case has been cancelled.**Note:** Only invoice cases in the Draft state can be cancelled.

</td></tr></tbody>
