# Mobile security practices

Mobile security practices include mobile-specific system properties, attachment control, password reinforcement, security patching, and controlling shared data.

-   **[Security controls](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/mobile/mobile-platform/sg-mobile-security-controls.md)**  
Configure security controls to restrict copy/paste, enforce PIN, or block attachment functionality.
-   **[Security patching](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/mobile/mobile-platform/sg-mobile-security-patching.md)**  
In the event a security patch is needed, the mobile development team aligns with standard SDLC properties in order to patch.
-   **[User data collection](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/mobile/mobile-platform/sg-mobile-security-user-data.md)**  
ServiceNow mobile apps do not specifically collect any user data.
-   **[Shared data](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/mobile/mobile-platform/sg-mobile-security-shared-data.md)**  
ServiceNow mobile apps communicate with a third party software for app crash reporting. No customer information is shared.
-   **[Incident reporting](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/mobile/mobile-platform/sg-mobile-security-incidents.md)**  
ServiceNow mobile app issues should be reported through the standard support channels. You can report incidents by contacting Customer Service and Support.
