# Configure cards in Mobile App Builder

Use cards to visually display information from different records by mapping data to the UI elements defined in a card template. UI rules can also be used to define conditional styling on cards.

## Before you begin

Role required: admin or delegated developer

## Procedure

1.  Navigate to **All** &gt; **System Mobile** &gt; **Mobile App Builder**.

    The Mobile App Builder opens in a new browser tab and displays the application scope selection screen.

2.  Search for the application scope you’re working in and then select the name of the application screen.

    The Mobile App Builder categories home screen displays.

3.  Select **Cards &amp; icons** in the menu, and then select **New**.

4.  Select **Card** from the displayed menu, and then select **Continue**.

5.  Complete the following fields as needed.

    |Field|Properties|
    |-----|----------|
    |Properties|
    |Name|Name for the card.|
    |Description|Additional information about your card.|
    |Active|Determines if this card is visible in your mobile app.|
    |Data|
    |Table|Table from which the card gets its data.|
    |Card template|Choose a card template that will create the UI layout for your card.|
    |Card fields|Add additional fields from the card's table that aren’t included in the card template.|
    |UI rules|Determine how the mobile interface will change or which fields will update when certain conditions are met.|
    |Advanced scripting of UI rule conditions|Toggle to allow for variables not present in the card template to be used in the UI rule conditions of your mobile card.|

6.  Select **Save**.

7.  Select **Open in Mobile Card Builder** to open the Mobile Card Builder view.

    The Mobile Card Builder view displays.

8.  Edit the appearance of your card and select **Save** once you’re finished.

9.  Close the Mobile Card Builder view to return to your Mobile App Builder screen.
