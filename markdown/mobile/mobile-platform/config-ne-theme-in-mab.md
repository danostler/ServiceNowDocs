# Assign a Next Experience theme in the Mobile App Builder

Use the Mobile App Builder to assign your Next Experience themes.

## Before you begin

Role required: admin

To use Next Experience theming, you must have Next Experience enabled on your instance. For activation instructions and more information on Next Experience, see [Exploring themes in Next Experience](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/platform-user-interface/configure-user-experiences/next-experience-theming.md).

**Note:** If you are a new customer starting with the Zurich release, Next Experience is active by default on your instance. No activation is required, and your mobile apps use the **Coral** theme automatically.

## Procedure

1.  Open the Mobile App Builder by navigating to **All** &gt; **System** &gt; **Mobile App Builder**.

2.  Select the application that contains the mobile app you want to theme.

    For example, **Now Mobile**.

3.  In the menu on the left side of the screen, select **Mobile app configs**

4.  Select the mobile app you want to theme from the list.

5.  In the **Mobile theme** section, select a theme from the list.

    With Next Experience enabled, your instance uses the **Coral** by default. If you want to continue to use this theme, you do not need to update this field. You will only need to change this field if you:

    -   Have a custom theme that you want to apply to the app.
    -   Want to apply the former **Polaris** theme to the app.
    -   Had previously opted out of Next Experience.
    For information on creating your own themes, see [Exploring themes in Next Experience](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/platform-user-interface/configure-user-experiences/next-experience-theming.md).

6.  Click the **Save** button in the upper right corner of the screen to save your changes.

    Your new theme is applied immediately, logged in users will see the new theme the next time the app refreshes.
