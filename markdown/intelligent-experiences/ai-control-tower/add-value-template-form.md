# Add value template form

Add value template form fields and descriptions.

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td colspan="2">

Calculation builder

</td></tr><tr><td>

Persona

</td><td>

The user role for whom the value is being calculated, for example, Agent, Requester.

</td></tr><tr><td>

Usage

</td><td>

The usage indicator that you want to associate the value template with.

</td></tr><tr><td>

Time value type

</td><td>

The type of time-based metric used in the calculation, for example, Constant or Indicator. Choose a value that aligns with your performance goals.

</td></tr><tr><td>

Time constant

</td><td>

A fixed time value \(in hours, minutes, or days\) used as a baseline or threshold in the calculation. Useful for comparisons or normalization.

</td></tr><tr><td>

Time indicator

</td><td>

The time indicator that you want to associate the value template with.

</td></tr><tr><td>

Acceptance rate type

</td><td>

The type of acceptance metric to include, such as Indicator or Constant.

</td></tr><tr><td>

Acceptance rate constant

</td><td>

A fixed percentage or ratio used to represent an expected or target acceptance rate in the value calculation.

</td></tr><tr><td colspan="2">

Attributes

</td></tr><tr><td>

Template name

</td><td>

A unique and descriptive name for the value template. This name appears in the list of templates and should clearly indicate its purpose.

</td></tr><tr><td>

Value template category

</td><td>

The categorization of the template based on its use case, for example, Productivity, Efficiency, Accuracy. Helps in organizing and filtering templates.

</td></tr><tr><td>

Department

</td><td>

The business unit or department associated with the value template. This helps in aligning the template with organizational goals.

</td></tr><tr><td>

Description

</td><td>

A brief explanation of what the value template does, including its purpose and how it contributes to measuring value.

</td></tr><tr><td colspan="2">

Calculation output

</td></tr><tr><td>

Instance

</td><td>

The instance where the value template is applied. This field is useful for managing value templates across instances.**Note:** You can validate a formula in a non-production instance before publishing it to a production instance.
