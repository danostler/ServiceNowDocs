# Metric Threshold form fields

Describes the fields in the Metric Threshold form.

<table id="table_xp1_3c5_txb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Name

</td><td>

Name for the threshold.

</td></tr><tr><td>

Threshold type

</td><td>

Type of threshold. The available options are:-   Static: A fixed value used as a limit for tracking a metric.
-   Dynamic: Percentage-based thresholds where variance is calculated based on previous period data

For more information, see [Thresholds for metrics](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/environmental-social-governance/operational-sustainability-management/thresholds-for-metrics.md)

</td></tr><tr><td>

Color

</td><td>

Color that represents this  threshold level. The available color list  can be customized, and each threshold level must have a unique color. For information on how to customize colors, see [Customize threshold colors](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/environmental-social-governance/operational-sustainability-management/customize-threshold-colors.md).

</td></tr><tr><td>

Valid from

</td><td>

Date from which the threshold configuration is valid.

</td></tr><tr><td>

Valid until

</td><td>

Date until the threshold configuration is valid.

</td></tr><tr><td>

Active

</td><td>

Indicates whether the threshold is active. 

</td></tr><tr><td>

Enable threshold breach monitoring

</td><td>

Indicates whether the system prompts for justification when a threshold is breached.

</td></tr><tr><td>

Condition

</td><td>

Conditions that determine when the threshold applies.

</td></tr><tr><td>

Actions

</td><td>

Actions to execute when the threshold condition is satisfied. By default, Send email is available. Additional actions can be created using decision tables.

</td></tr></tbody>
