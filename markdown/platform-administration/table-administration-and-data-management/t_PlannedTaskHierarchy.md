# Planned task hierarchy

The **Task Hierarchy** tool available for Planned Task displays the relationship between parent and child planned tasks.

## Before you begin

Role required: admin

## About this task

Out-of-box, the **Task Hierarchy** tool is available in both Project and Release Management.

Different **Planned Task** tables have different UI actions to launch the task hierarchy:

-   To view the hierarchy of a Project, navigate to the use the **Task hierarchy** context menu action.
-   To view the hierarchy of a Product in Release v2, navigate to the product and click the **Product hierarchy** related link.
-   To view the hierarchy of a Release in Release v2, navigate to the release and click the **Release hierarchy** related link.

\[Omitted image "Rm2Heirarchy.png"\] Alt text: Release hierarchy

The Task Hierarchy can be added to any planned task table by:

## Procedure

1.  Navigating to **System UI** &gt; **UI Actions**.

2.  Selecting one of the existing Task Hierarchy UI Actions \(for example, **Task hierarchy** if Project Management is activated\).

3.  Change the table to the desired table and rename the UI Action if appropriate, and insert.

    The hierarchy should now be available as a UI Action on the form of the new table.
