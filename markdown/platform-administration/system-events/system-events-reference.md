# System events reference

Reference topics provide information about system events.

-   **[Event logs](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown)**  
The event log records all system events that occur within the ServiceNow AI Platform.
-   **[Event states](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/platform-administration/system-events/r_EventStates.md)**  
The event state describes where in the life cycle the event is.
-   **[The incident events business rule](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/platform-administration/system-events/r_BuildAScript.md)**  
The incident events business rule comes with the system and defines a number of events that can be triggered by different actions in the Incident table.
-   **[Global events](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/platform-administration/system-events/r_GlobalEvents.md)**  
Your instance has a global function called global\_events\(\) that triggers from a business rule when certain conditions occur.
-   **[Script actions](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/platform-administration/system-events/r_ScriptActions.md)**  
You can use script actions to create server-side scripts that perform a variety of tasks, such as modifying a configuration item \(CI\), or managing failed login attempts. Script actions are triggered by events only.
-   **[Sample scripts from the change events business rule](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/platform-administration/system-events/r_ChangeEventsSampleScripts.md)**  
Several scripts are found in the baseline change events business rule.
