# Customize system notifications

Create custom notifications, which are subscriptions to notifications of importance to you, and apply conditions that control specific content included in your custom notification. You can also enable or disable the channels for delivery.

## Before you begin

Role required: none

## About this task

**Note:** Custom notifications for Next Experience do not support the setting for the affected record the notification is based on. For notifications that require an associated affected record, see [Create personal notifications](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-administration/ai-platform-administration/create-personal-notifications.md).

## Procedure

1.  From your user profile, navigate to **Preferences** &gt; **Notifications** and then select **Advanced Preferences**.

2.  Select the **Custom Notifications** tab.

3.  Select **Create notification**.

4.  Select the notification you want to receive.

    1.  Enter a name for the notification in the **Personal notification name** field.

    2.  Select the system notification you want to receive from the **Notification** drop-down.

    3.  Select **Next**.

5.  Select the channels from the delivery channels through which you want to receive the notifications.

6.  Select **Next**.

7.  Set a schedule or conditions for receiving notifications.

    1.  Choose to have notifications delivered only during weekday working hours by selecting **Monday thru Friday \(9 to 5\)**.

    2.  Set a filter with as many conditions as desired to limit the notifications you receive.

        For example, you can choose to be notified only when an incident communication plan is created for a specific configuration item.

        **Note:** All conditions are set to **\[Active\]\[is\]\[true\]** by default.

8.  Select **Save**.
