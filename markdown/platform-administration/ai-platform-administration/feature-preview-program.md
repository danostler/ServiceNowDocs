# Feature Preview Program

The Feature Preview Program provides access to pre-release capabilities on your instance. You can activate, test, and provide feedback on individual features before they are generally available.

When a pre-release feature is added to your instance, you can access the Feature Preview Program to review feature details, activate the feature for testing, and provide feedback. You're fully in control of the feature preview experience and can turn features on or off at any time.The program displays all available pre-release features with information about what each feature includes and how it works. Your feedback on pre-release features helps identify issues, and refine functionality.

Each feature is made available on a per-feature basis, you must explicitly activate each feature you want to use, and you can deactivate any feature at any time. You can activate or deactivate pre-release features. For each feature, you can review the following information:

-   Feature name
-   Feature overview describing what the feature does
-   Details about what is included in the feature

## How the program works

Before a feature appears on your Feature Preview Program page, the product team that owns the feature works directly with a targeted set of customers to confirm their participation. Once your organization agrees to participate, the feature is released on your instance and is visible on the Feature Preview Program page.

When a new pre-release feature is added to your instance, a toggle notification appears to alert you. Selecting the notification takes you to the Feature Preview Program page, where you can review available features and choose which ones to activate.

Activation is per-feature. When you activate a feature, you provide consent for that specific feature only. Activating one feature does not affect other features listed on the page.

## What to consider before activating a feature

Before you activate a pre-release feature, review the following considerations.

-   **Environment:** Features activate directly on your production instance. Review the feature details carefully and confirm that activation is appropriate for your environment before proceeding.
-   **Support:** If you experience issues with a pre-release feature, you can contact the product team directly through the Feature Preview Program page.
-   **Access control:** The admin role is required to activate features. Review who in your organization has the authority to accept the consent agreement on behalf of your organization. If you're unsure, consult your platform owner or legal team before proceeding.
-   **Reversibility:** You can deactivate a feature at any time. However, deactivating a feature disables its functionality but does not remove all artifacts it installed on your instance.

## Accessing the Feature Preview Program

You can access the Feature Preview Program in the following ways:

-   **Notification**

    When a pre-release feature is added to your instance, you receive a notification indicating that a pre-release capability is available. Select the notification to open the Feature Preview Program page.

-   **All menu**

    Navigate to **All** &gt; **Feature Preview Program**.


## Providing feedback

Feedback is an important part of the Feature Preview Program. When you test pre-release features, your feedback helps improve the feature before it reaches general availability. You can provide feedback on feature functionality, usability, and performance. Your feedback helps identify issues, refine features, and confirm that the features meet customer needs when they are released.

You can activate or deactivate pre-release features from this page. After activating and testing a feature, provide feedback to the product team.

-   **[Activate a pre-release feature](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-administration/ai-platform-administration/activate-prerelease-feature.md)**  
 Activate a pre-release feature on the instance so your users can try it out and provide feedback to the product team.
-   **[Deactivate a pre-release feature](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-administration/ai-platform-administration/deactivate-prerelease-feature.md)**  
Deactivate a pre-release feature if it is not working as expected or if you no longer need the feature.
