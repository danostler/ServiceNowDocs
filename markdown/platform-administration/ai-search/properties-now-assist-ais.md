# Now Assist in AI Search properties

System properties that you can use to modify behavior for Now Assist in AI Search Genius Results.

These properties are available for Now Assist in AI Search.

**Note:** To open the System Property \[sys\_properties\] table's list view, select **All**, enter `sys_properties.list` in the **Filter** field, and press **Enter**.

sn\_ais\_assist.semantic\_cache\_mode

</td><td>

Operational mode for the Now Assist Q&amp;A Genius Results caches. Use this mode to control how AI Search uses the caches and the Now LLM Service when looking for Now Assist Q&amp;A Genius Result answers. For details about how AI Search caches Now Assist Q&amp;A Genius Result answers, see [Caching for Now Assist Q&amp;A Genius Results](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/platform-administration/ai-search/caching-now-assist-q-a-gr.md).

-   Type: string
-   Default value: off
-   Supported values:
    -   **off**: Use the first-level cache and the Now LLM Service to find Now Assist Q&amp;A Genius Result answers.

AI Search looks in the first-level cache for Now Assist Q&amp;A Genius Result answers that exactly match your search query and knowledge article result. If it doesn't find a matching answer, it sends your query and Knowledge article sys\_id to the Now LLM Service for answer generation.

Now Assist Q&amp;A Genius Result answers generated by the Now LLM Service populate the first-level cache.

When using Dynamic Translation, AI Search bypasses the caches and queries the Now LLM Service to generate an answer for the Now Assist Q&amp;A Genius Result.

    -   **offline**: Use the first-level and second-level caches to find Now Assist Q&amp;A Genius Result answers. Don't submit queries to the Now LLM Service.

AI Search looks in the first-level cache for Now Assist Q&amp;A Genius Result answers that exactly match your search query and knowledge article result. If it doesn't find a matching answer, it uses semantic vector search to look for answers that match the meaning of your query in the second-level cache. If no cached answers match your query and knowledge article result, AI Search returns no answer for the Now Assist Q&amp;A Genius Result.

Now Assist Q&amp;A Genius Result answers found in the second-level cache populate the first-level cache.

When using Dynamic Translation, AI Search bypasses the caches and returns no answer for the Now Assist Q&amp;A Genius Result.

    -   **online**: Use the first-level and second-level caches and the Now LLM Service to find Now Assist Q&amp;A Genius Result answers.

AI Search looks in the first-level cache for Now Assist Q&amp;A Genius Result answers that exactly match your search query and knowledge article result. If it doesn't find a matching answer, it uses semantic vector search to look for answers that match the meaning of your query in the second-level cache. If no cached answers match your query and knowledge article result, AI Search submits the query and article sys\_id to the Now LLM Service for answer generation.

Now Assist Q&amp;A Genius Result answers generated by the Now LLM Service populate the first-level and second-level caches.

When using Dynamic Translation, AI Search bypasses the caches and queries the Now LLM Service to generate an answer for the Now Assist Q&amp;A Genius Result.

-   Location: Not present in the System Property \[sys\_properties\] table by default. To override the default value, add this property by following the steps in [Add a system property](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/platform-administration/t_AddAPropertyUsingSysPropsList.md).

</td></tr><tr><td>

sn\_ais\_assist.u\_question\_regex

</td><td>

Case-insensitive Java regular expression pattern that defines searches eligible for Now Assist Q&amp;A answer generation. As an example, the pattern `^(what|where|when|why|who|whom|which|how)\b` restricts eligibility to search queries that begin with one of the specified keywords. When this system property has a value defined, only searches in supported applications that match the specified pattern are eligible for Now Assist Q&amp;A answer generation.

 When no value is defined, all searches in supported applications are eligible for Now Assist Q&amp;A answer generation.

 -   Type: string
-   Default value: none
-   Location: System Property \[sys\_properties\] table

 To learn about Java regular expression pattern syntax, see [the Javadoc for the java.regex.util.Pattern class](https://docs.oracle.com/en/java/javase/21/docs/api/java.base/java/util/regex/Pattern.html).

</td></tr><tr><td>

sn\_ais\_assist.u\_sanitize\_html

</td><td>

Option to remove HTML tags from knowledge article text before sending to the Now LLM Service for Now Assist Q&amp;A answer generation. -   Type: true \| false
-   Default value: false
-   Location: Not present in the System Property \[sys\_properties\] table by default. To override the default value, add this property by following the steps in [Add a system property](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/platform-administration/t_AddAPropertyUsingSysPropsList.md).
