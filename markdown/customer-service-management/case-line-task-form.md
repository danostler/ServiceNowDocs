# Case line task form

The case line task form displays details about a case line task associated with a case line item on a case record.

<table id="table_vyg_gzg_lvb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Number

</td><td>

The automatically generated record number. The prefix for numbers from the Case Line Task table is CSLTASK.

</td></tr><tr><td>

Parent case

</td><td>

The parent case of the case line item.

</td></tr><tr><td>

Account

</td><td>

The name of the company that the parent case is opened for.

</td></tr><tr><td>

Contact

</td><td>

The name of the customer contact for the parent case.

</td></tr><tr><td>

Case line

</td><td>

The number of the case line item that the task is created for.

</td></tr><tr><td>

State

</td><td>

The state of the case line task:-   Open
-   Closed

</td></tr><tr><td>

Priority

</td><td>

The priority of the case line task:-   1 - Critical
-   2 - High
-   3 - Moderate
-   4 - Low

</td></tr><tr><td>

Assignment group

</td><td>

The assigned customer service agent group.

</td></tr><tr><td>

Assigned to

</td><td>

The agent assigned to the case line task.

</td></tr><tr><td>

Subject

</td><td>

The subject of the case line task.

</td></tr><tr><td>

Description

</td><td>

A longer, more detailed description of the case line task.

</td></tr><tr><td>

Work notes list

</td><td>

A list of users who are interested in the case task. These users receive a notification when work notes are added to the case task.

</td></tr><tr><td>

Work notes

</td><td>

Information about the case task and the steps taken to resolve it. Work notes are posted to the activity stream and are visible only to internal users.

</td></tr><tr><td>

Additional comments

</td><td>

Customer-viewable comments and information about the case task. Additional comments are posted to the activity stream.

</td></tr></tbody>
</table>
