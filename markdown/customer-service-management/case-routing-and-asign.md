# Case routing and assignment

Set up Advanced Work Assignment \(AWA\) to automatically route cases to the appropriate customer service agents in CSM Configurable Workspace.

|Topic|Description|
|-----|-----------|
|[Exploring Advanced Work Assignment](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/conversational-interfaces/advanced-work-assignment/awa-overview.md)|Learn more about the AWA tool, including concepts such as work assignment and assignment rules, service channels, work items and work item queues, and agent capacity.|
|[AWA for CSM](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/customer-service-management/awa-csm-overview.md)|Use AWA to automatically assign work to your agents based on availability, capacity, and skills. AWA pushes work to qualified agents using work item queues, routing conditions, and assignment criteria that you define. Agents see their assignments in their CSM Configurable Workspace inbox.|
|[Set up an Advanced Work Assignment service channel for assigning case tasks](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/customer-service-management/configure-awa-channel-case-tasks.md)|Set up an Advanced Work Assignment \(AWA\) service channel that you can use to assign case tasks to available users.|
|[Configure AWA queues for the proxy contact role](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/customer-service-management/config-awa-queue-for-proxy-contact.md)|Modify the Advanced Work Assignment queues and add routing conditions that support the [proxy contact](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/zurich/markdown/zurich/customer-service-management/employee-create-case-for-customer.md) role. This role enables employees to create cases for customer accounts and contacts. Employees can also be proxy case contacts on behalf of customers.|
