# Task Intelligence Admin Console

Use the Task Intelligence Admin Console to create, train, and deploy machine learning models that predict different types of information for case and interaction records.

\[Omitted image "task-intel-admin-console-3.png"\] Alt text: Task Intelligence Admin console displaying prediction metrics, a detailed list of machine learning models, model templates, and access to related applications.

From the Admin Console, you can set up predictive models, preview the agent's experience, view when the models are active, and track model performance.

The Admin Console provides tools that you can use to create and implement machine learning models in just a few steps. Each model follows a six-step process.

<table id="table_rt4_2hv_c5b"><tbody><tr><td>

**1.** Select a model to use as a starting point.

</td><td>

Select the model based on what you want the model to do. For example:-   Use the field prediction model to make field value predictions that you can use to categorize cases and interactions.
-   Use the case sentiment model to predict sentiment at any point from case creation to case resolution.

</td></tr><tr><td>

**2.** Define the purpose of the model.

</td><td>

Tell the model when you want it to make predictions and what you want it to predict. For example, predict the category and priority when a new case is created.

</td></tr><tr><td>

**3.** Select the data used to train the model.

</td><td>

Train the model using selected data, such as the text in the case short description and description, so it can learn patterns in the data. Then test the model to see how well it works.

</td></tr><tr><td>

**4.** Assess the model's results.

</td><td>

View test results to see how your well the model performed. These results indicate how a model will perform after being deployed.

</td></tr><tr><td>

**5.** Select preferences for prediction results.

</td><td>

Add predictions directly to record fields, show predictions as recommendations, or monitor predictions in the background.

</td></tr><tr><td>

**6.** Deploy the model.

</td><td>

Review your selections and start using the model.

</td></tr></tbody>
